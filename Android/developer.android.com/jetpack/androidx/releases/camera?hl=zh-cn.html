声明依赖项

如需添加 CameraX 的依赖项，您必须将 Google Maven 代码库添加到项目中。有关详情，请阅读 Google 的 Maven 制品库。

在应用或模块的 build.gradle 文件中添加所需工件的依赖项：

Groovy

dependencies {
  // CameraX core library using the camera2 implementation
  def camerax_version = "1.5.0"
  // The following line is optional, as the core library is included indirectly by camera-camera2
  implementation "androidx.camera:camera-core:${camerax_version}"
  implementation "androidx.camera:camera-camera2:${camerax_version}"
  // If you want to additionally use the CameraX Lifecycle library
  implementation "androidx.camera:camera-lifecycle:${camerax_version}"
  // If you want to additionally use the CameraX VideoCapture library
  implementation "androidx.camera:camera-video:${camerax_version}"
  // If you want to additionally use the CameraX View class
  implementation "androidx.camera:camera-view:${camerax_version}"
  // If you want to additionally add CameraX ML Kit Vision Integration
  implementation "androidx.camera:camera-mlkit-vision:${camerax_version}"
  // If you want to additionally use the CameraX Extensions library
  implementation "androidx.camera:camera-extensions:${camerax_version}"
}

相机版本 1.5

版本 1.5.0

2025 年 9 月 10 日

发布了 androidx.camera:camera-*:1.5.0。版本 1.5.0 中包含这些提交内容。

自 1.4.0 以来的重要变更：

  • 高速和慢动作录制：只需少量代码即可轻松集成高速 (120/240 fps) 和慢动作视频录制功能。如需了解详情，请参阅 Recorder#getHighSpeedVideoCapabilities(CameraInfo) 和 HighSpeedVideoSessionConfig。
  • SessionConfig 和 FeatureGroup API：新的 SessionConfig API 可让您安全地同时配置相机会话并启用多项功能，包括 HLG (HDR)、UltraHDR、60 FPS 和预览防抖。FeatureGroup API 可让您安全地同时配置相机会话并启用多项功能，包括 HLG (HDR)、UltraHDR、60 FPS 和预览防抖。您还可以设置具有优先级的首选功能组，让 CameraX 确定最佳支持组合。如需了解详情，请参阅 SessionConfig.Builder#setPreferredFeatureGroup、SessionConfig.Builder#setRequiredFeatureGroup 和 CameraInfo#isFeatureGroupSupported(SessionConfig)。
  • 确定性帧速率 API：使用 CameraInfo.getSupportedFrameRateRanges(sessionConfig) 查询和 SessionConfig.setExpectedFrameRateRange 设置精确且受支持的帧速率，从而解决之前 setTargetFrameRate 存在的限制。
  • 相机扩展程序：现在，扩展程序支持 UltraHDR 格式。勾选 ImageCapture.getImageCaptureCapabilities(cameraInfo).getSupportedOutputFormats()，然后在 ImageCapture.Builder.setOutputFormat 中启用该功能。现在，当扩展程序处于启用状态时，系统会反映变焦比和预览防抖功能。
  • 手电筒强度：使用 CameraControl.setTorchStrengthLevel 调整手电筒强度。
  • 弱光增强模式：您可以通过以下 API 启用弱光增强模式(CONTROL_AE_MODE_ON_LOW_LIGHT_BOOST_BRIGHTNESS_PRIORITY)：请参阅 CameraInfo#isLowLightBoostSupported、CameraInfo#getLowLightBoostState 和 CameraControl#enableLowLightBoostAsync。
  • 视频拍摄：现在，当录制期间存储空间不足时，系统会触发 VideoRecordEvent.Finalize.ERROR_INSUFFICIENT_STORAGE。PendingRecording.withAudioEnabled(boolean initialMuted) 允许控制音频录制的初始静音状态。
  • 图片拍摄：在 ImageCapture 中支持 DNG (RAW) 和 JPEG + DNG (RAW) 格式。检查 ImageCaptureCapabilities(CameraInfo).getSupportedOutputFormats() 是否支持 RAW。使用具有多个 OutputFileOptions 的重载 takePicture API 进行 RAW+DNG 拍摄。
  • 图片分析：支持 ImageAnalysis 中的 NV21 格式。使用 ImageAnalysis.Builder.setOutputImageFormat(OUTPUT_IMAGE_FORMAT_NV21) 启用该功能。

新功能

宽高比和分辨率选择方面的变更

CameraX 的目标是成功初始化相机会话。这意味着，CameraX 为了能够启动捕获会话（这是它的首要目标），会根据设备显示能力在分辨率/宽高比上做出让步，因此可能无法完全按照请求的方式显示。可能原因如下：

  • 设备不支持请求的分辨率
  • 兼容性问题，例如在旧版设备上，必须使用某些分辨率才能正确运行
  • 在某些设备上，某些格式仅在某些宽高比下可用
  • 对于 JPEG 或视频编码，首选“最近的 mod16”。请参阅 CameraCharacteristics#SCALER_STREAM_CONFIGURATION_MAP。

尽管 CameraX 会创建并管理会话，但您应始终在代码中检查用例输出所返回的图像大小，并进行相应调整。

对分辨率和宽高比设置进行了一些更改，旨在使 API 更清晰明了：

  • 现在，如果未设置宽高比，则预览用例将采用默认的 4:3 宽高比。
  • 当 CameraX 根据设备显示能力在内部考虑对所请求的分辨率和宽高比进行更改时，它首先会尝试保持相同的宽高比（由任意 setTargetAspectRatio 或 setTargetResolution 调用确定）
  • 分辨率的“最近的 mod16”版本被视为相同的宽高比。

ImageAnalysis 非阻塞模式

  • 现在，ImageReaderMode.ACQUIRE_LATEST_IMAGE 的行为是非阻塞行为。它会获取队列中的最新图像，但连续舍弃未使用的图像，以免相机管道发生阻塞。
  • 分析器可以无限期保存单个图像，而不会阻塞管道。
  • 如果应用提供了一个执行器，该执行器继而发生阻塞，则 ImageAnalysis 用例会发生阻塞。
  • 内部设置的默认执行器会表现为非阻塞执行器。

  • 修复了未针对用例调用 setLensFacing 而导致 java.lang.IllegalArgumentException: Unable to get camera ID for use case 的问题。

  • 修复了打开相机时应用进入后台运行的问题
  • 修复了 API <25 上的问题，移除了错误 checkAndUpdateEglState: invalid current EGLDisplay

    preview.setSurfaceProvider(previewView.previewSurfaceProvider)
cameraProvider.bindToLifecycle(lifecycleOwner, cameraSelector, preview)

    val camera = cameraProvider.bindToLifecycle(lifecycleOwner, cameraSelector, preview)
previewView.preferredImplementationMode = ImplementationMode.TEXTURE_VIEW
preview.setSurfaceProvider(previewView.createSurfaceProvider(camera.cameraInfo))
