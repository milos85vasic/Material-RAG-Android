androidx.credentials:credentials:1.5.0-alpha03 และ androidx.credentials:credentials-play-services-auth:1.5.0-alpha03 เปิดตัวแล้ว เวอร์ชันนี้ได้รับการพัฒนาในสาขาภายใน

ฟีเจอร์ใหม่

  • ขอแนะนำฟีเจอร์ใหม่ "กู้คืนข้อมูลเข้าสู่ระบบ" ระบบจะใช้ข้อมูลเข้าสู่ระบบสำหรับการกู้คืนเพื่อกู้คืนข้อมูลเข้าสู่ระบบของผู้ใช้จากอุปกรณ์เครื่องก่อนหน้าไปยังอุปกรณ์ Android เครื่องใหม่ การสร้าง RestoreCredential สำหรับผู้ใช้จะโอนข้อมูลเข้าสู่ระบบไปยังอุปกรณ์ใหม่ของผู้ใช้โดยอัตโนมัติ หากผู้ใช้เลือกแอปที่จะโอนจากอุปกรณ์เครื่องเก่าในขั้นตอนการตั้งค่า

การเปลี่ยนแปลง API

  • เราได้เพิ่มคลาสใหม่สำหรับการขอข้อมูลเข้าสู่ระบบการกู้คืน
  • ข้อมูลเข้าสู่ระบบประเภทใหม่ RestoreCredential ที่สามารถกู้คืนข้อมูลเข้าสู่ระบบในอุปกรณ์ใหม่ได้
  • CreateRestoreCredentialRequest เพื่อสร้าง RestoreCredential ใหม่
  • GetRestoreCredentialOption สำหรับการดึงข้อมูล RestoreCredential
  • ClearCredentialStateRequest สามารถแก้ไขเพื่อล้าง RestoreCredential ได้

การแก้ไขข้อบกพร่อง

  • เพิ่ม RestoreCredential API ใหม่เพื่อวัตถุประสงค์ในการกู้คืนแอป (If2d40)

เวอร์ชัน 1.5.0-alpha01

29 พฤษภาคม 2024

androidx.credentials:credentials:1.5.0-alpha01 และ androidx.credentials:credentials-play-services-auth:1.5.0-alpha01 เปิดตัวแล้ว เวอร์ชันนี้ได้รับการพัฒนาในสาขาภายใน

ฟีเจอร์ใหม่

ประสบการณ์ UI รองสำหรับการเลือกข้อมูลเข้าสู่ระบบ: ตอนนี้นักพัฒนาแอปที่เรียกใช้ API ของเครื่องมือจัดการข้อมูลเข้าสู่ระบบในขณะที่ลงชื่อเข้าใช้เพื่อแสดงตัวเลือกต่อผู้ใช้จะใช้ API ใหม่เพื่อเชื่อมโยง GetCredentialRequest เดียวกันกับมุมมองที่กำหนดได้ เช่น ช่องชื่อผู้ใช้หรือรหัสผ่าน

จากนั้น เมื่อผู้ใช้โฟกัสที่มุมมองใดมุมมองหนึ่งเหล่านี้ ระบบจะส่งคำขอที่เกี่ยวข้องไปยังเครื่องมือจัดการข้อมูลเข้าสู่ระบบ ระบบจะรวบรวมข้อมูลเข้าสู่ระบบที่ได้จากผู้ให้บริการต่างๆ และแสดงในฟีเจอร์ป้อนข้อความอัตโนมัติ เช่น UI รอง เช่น แป้นพิมพ์หรือคำแนะนำแบบดร็อปดาวน์ ดังนั้นเมื่อใช้ API ทั้งหมดร่วมกัน ระบบจะแสดงตัวเลือกแก่ผู้ใช้ก่อน และหากผู้ใช้ปิดตัวเลือกและแตะช่องใดช่องหนึ่งที่กล่าวถึงข้างต้น ระบบจะแสดงคำแนะนำจากแป้นพิมพ์/เมนูแบบเลื่อนลง

การเปลี่ยนแปลง API

  • PendingGetCredentialRequest คลาสที่รับ GetCredentialRequest (ที่มีอยู่แล้ว) และการเรียกกลับที่จะเรียกใช้กับ GetCredentialResponse (ที่มีอยู่แล้ว) เมื่อพร้อมใช้งานแบบไม่พร้อมกัน
  • API ตัวตั้งค่าส่วนขยายใหม่สำหรับคลาส View ของ Android ซึ่งอนุญาตให้ตั้งค่าอินสแตนซ์ของ PendingGetCredentialRequest การใช้ API นี้จะเตรียมมุมมองที่ระบุไว้เพื่อให้เมื่อผู้ใช้แตะมุมมองนั้น คำแนะนำข้อมูลเข้าสู่ระบบจะแสดงในประสบการณ์ UI รอง เช่น คำแนะนำแป้นพิมพ์/เมนูแบบเลื่อนลง

  • นำการใช้งาน isAtLeastU() API เวอร์ชันทดลองออก (Ie9117, b/289269026)

  • เพิ่มคำอธิบายประกอบ VisibleForTest (I5467a)
  • เพิ่มคำอธิบายประกอบ VisibleForTest (Idf57a)
  • นำ API สำหรับการทดสอบเท่านั้นออก (Idcc05)
  • แสดงคลาสรายการของผู้ให้บริการไปยังระดับ API ที่ต่ำกว่า (I2e00a)
  • เพิ่ม API ทดสอบ (Id6b9e)

การแก้ไขข้อบกพร่อง

  • เพิ่ม API ทดสอบ (I0d243)
  • เพิ่ม API การทดสอบใหม่ (I6fa12)
  • แสดงการเลือกอัตโนมัติสำหรับคำขอสร้าง (I84eee)
  • ทำให้ข้อผิดพลาดในการเข้ารหัส JSON มีรายละเอียดมากขึ้น (I7a865)
  • รายงานข้อผิดพลาดของนักพัฒนาซอฟต์แวร์อย่างเหมาะสมเมื่อพารามิเตอร์บริบทที่ไม่มีกิจกรรม (/I20dd7, b/288288940)
  • การแยกวิเคราะห์ข้อยกเว้นที่แก้ไขแล้วสำหรับข้อยกเว้นที่ส่งคืนจากผู้ให้บริการ (Iaa2af,I0d243,I55151)
  • ปรับปรุงเอกสารประกอบสำหรับ toSlice

เวอร์ชัน 1.2.0-alpha02

8 มีนาคม 2023

androidx.credentials:credentials:1.2.0-alpha02 และ androidx.credentials:credentials-play-services-auth:1.2.0-alpha02 เปิดตัวแล้ว พัฒนาจากสาขาภายใน

หมายเหตุ: เวอร์ชันนี้จะคอมไพล์กับ Android 14 Developer Preview 2 SDK เท่านั้น

การเปลี่ยนแปลง API

  • เปิดใช้การทดสอบคลาสคำขอของผู้ให้บริการโดยทำให้ตัวสร้างเป็นแบบสาธารณะ
  • กำหนดให้ต้องมีไอคอนในคลาสรายการทั้งหมด อย่างไรก็ตาม หากผู้ให้บริการเอกสารรับรองไม่ได้ระบุไอคอน ไลบรารีนี้จะมีไอคอนสำรอง
  • อนุญาตให้ผู้ให้บริการข้อมูลเข้าสู่ระบบตั้งค่ารายการการดำเนินการตรวจสอบสิทธิ์หลายรายการ และตั้งชื่อสำหรับแต่ละรายการ
  • นำคลาสคำขอที่มีสิทธิ์ทั้งหมดออก ตอนนี้ผู้ให้บริการสามารถรับต้นทางจากคลาส android.service.credentials.CallingAppInfo ได้อย่างง่ายดาย และไม่จำเป็นต้องจัดการคลาสคำขอพิเศษสำหรับการเรียกที่ได้รับสิทธิ์ (การเรียกในนามของแอปอื่น)
