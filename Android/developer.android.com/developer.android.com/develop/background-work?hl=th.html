งานที่ทำอยู่เบื้องหลัง

งานในเบื้องหลังคืองานใดก็ตามที่แอปต้องดำเนินการเมื่อแอปไม่ปรากฏในเบื้องหน้า เช่น การดึงข้อมูลจากเซิร์ฟเวอร์หรือการประมวลผลรูปภาพ

การจัดการความซับซ้อนของการดำเนินการในเบื้องหลังเป็นกุญแจสำคัญในการสร้างแอปที่ "เป็นมิตร" นั่นคือ แอปพลิเคชันที่มีความยืดหยุ่น ตอบสนองได้ดี และมีประสิทธิภาพ ซึ่งทำงานได้ ดีในอุปกรณ์ทุกเครื่อง

เกี่ยวกับงานในเบื้องหลัง

เลือกตัวเลือกที่เหมาะสม

การเลือก API งานพื้นหลังที่เหมาะสมอาจเป็นเรื่องซับซ้อน หน้านี้จะให้คำแนะนำที่ชัดเจนเกี่ยวกับเวลาที่ควรใช้การทำงานแบบอะซิงโครนัส, WorkManager, บริการที่ทำงานอยู่เบื้องหน้า หรือตัวเลือกอื่น ดูว่า API ใดเหมาะกับ Use Case เฉพาะของคุณมากที่สุด

API และเครื่องมือที่ใช้กันโดยทั่วไปในการทำงานในเบื้องหลัง ได้แก่ WorkManager, บริการที่ทำงานอยู่เบื้องหน้า, การปลุก และการออกอากาศ แต่ละโซลูชันจะส่งผลต่ออายุการใช้งานแบตเตอรี่ของอุปกรณ์ของผู้ใช้แตกต่างกัน

WorkManager

WorkManager เป็นโซลูชันที่แนะนำสำหรับงานในเบื้องหลังที่ทำงานอย่างต่อเนื่องใน Android ดูวิธีใช้ไลบรารีที่มีประสิทธิภาพนี้เพื่อกำหนดเวลางานที่จะยังคงอยู่เมื่อแอปรีสตาร์ทและอุปกรณ์รีบูต รวมถึงทำความเข้าใจฟีเจอร์หลักในการจัดการข้อจำกัดและนโยบายการลองใหม่ที่ยืดหยุ่น
ดูข้อมูลเพิ่มเติม

บริการที่ทำงานอยู่เบื้องหน้า

บริการที่ทำงานอยู่เบื้องหน้าเป็นวิธีที่มีประสิทธิภาพในการเรียกใช้ฟังก์ชันที่ควรทำงานอย่างต่อเนื่องทันที ดูเวลาและวิธีประกาศ เปิดตัว และหยุดบริการที่ทำงานอยู่เบื้องหน้าพร้อมการแจ้งเตือนแบบต่อเนื่อง เพื่อให้มั่นใจว่าแอปจะทำงานที่สำคัญได้โดยไม่ถูกระบบปิด
ดูข้อมูลเพิ่มเติม

สัญญาณเตือนภัย

การปลุกเป็นเครื่องมือสำคัญในการกำหนดเวลางานตามเวลา หน้านี้จะอธิบายวิธีใช้ AlarmManager เพื่อตั้งเวลาการปลุกที่แน่นอนและไม่แน่นอนสำหรับงานที่ต้องทำงานในเวลาที่เฉพาะเจาะจงหรือภายในช่วงเวลาที่กำหนด แม้ว่าแอปจะไม่ได้ทำงานอยู่ก็ตาม นอกจากนี้ ยังครอบคลุมแนวทางปฏิบัติแนะนำในการจัดการการปลุกและการเพิ่มประสิทธิภาพเพื่อยืดอายุการใช้งานแบตเตอรี่

ออกอากาศ

ดูวิธีส่งและรับการออกอากาศเพื่อจัดการเหตุการณ์ของระบบและข้อความจากแอปอื่นๆ นอกจากนี้ คุณยังจะได้ทราบข้อมูลเกี่ยวกับตัวรับที่ลงทะเบียนตามบริบทและตัวรับที่ประกาศในไฟล์ Manifest รวมถึงข้อจำกัดของระบบที่ทันสมัยซึ่งมีผลกับตัวรับเหล่านั้น

คำแนะนำเบื้องต้นตามกรณีการใช้งาน

โซลูชันที่เหมาะสมจะแตกต่างกันไปตามกรณีการใช้งานของคุณ Android มี API เฉพาะงานมากมาย ซึ่งได้รับการเพิ่มประสิทธิภาพสำหรับสถานการณ์ที่เฉพาะเจาะจง และมักจะช่วยประหยัดพลังงานได้มากขึ้นและมีข้อจำกัดน้อยกว่าเมื่อใช้ WorkManager และบริการที่ทำงานอยู่เบื้องหน้า

ส่วนนี้แสดงกรณีการใช้งานที่พบบ่อยและโซลูชันที่แนะนำ

สถานที่

กรณีการใช้งานส่วนใหญ่จะกำหนดให้ต้องใช้ตำแหน่งเมื่อผู้ใช้มีส่วนร่วมกับแอปเท่านั้น อย่างไรก็ตาม หากแอปจำเป็นต้องเข้าถึงตำแหน่งในเบื้องหลัง ให้ใช้ Fused Location Provider API ใช้ประโยชน์จาก API นี้เพื่อรับตำแหน่งสุดท้ายที่ทราบซึ่งแคชไว้ หรือขออัปเดตตำแหน่งเป็นระยะ

วิดเจ็ต

อัปเดต GlanceAppWidgets อย่างมีประสิทธิภาพเมื่อแอปอยู่ในเบื้องหลังโดยใช้วิธีการอัปเดต

บลูทูธและอุปกรณ์ที่เชื่อมต่อ

ใช้บลูทูธพลังงานต่ำ (BLE) เพื่อสื่อสารกับอุปกรณ์ต่อพ่วงในเบื้องหลัง ดูวิธีสแกน เชื่อมต่อ และรักษาการเชื่อมต่อกับอุปกรณ์อย่างต่อเนื่อง พร้อมทั้งปฏิบัติตามข้อจำกัดระดับระบบและประหยัดแบตเตอรี่

การติดตามจำนวนก้าว

สำหรับการติดตามจำนวนก้าวในอุปกรณ์เคลื่อนที่ ให้พิจารณาอ่านจำนวนก้าวจาก Health Connect ซึ่งเป็นที่เก็บข้อมูล Android สำหรับข้อมูลสุขภาพและการออกกำลังกาย

การโอนข้อมูลที่เริ่มต้นโดยผู้ใช้

สำหรับงานในเบื้องหลังที่ผู้ใช้เริ่มต้นซึ่งเกี่ยวข้องกับการโอนข้อมูล (รวมถึงการดาวน์โหลดและการอัปโหลด) และมีจุดประสงค์เพื่อแจ้งให้ผู้ใช้ทราบความคืบหน้า ให้ใช้ประเภทงานการโอนข้อมูลที่เริ่มต้นโดยผู้ใช้ งานประเภทนี้ได้รับการเพิ่มประสิทธิภาพสำหรับการโอนข้อมูลที่ใช้เวลานาน และระบบจะให้สิทธิ์เข้าถึงก่อน

การเล่นสื่อ

ใช้ไลบรารี Media3 เพื่อรองรับการเล่นเบื้องหลังขณะที่ผู้ใช้มองไม่เห็นแอป

ทำความเข้าใจการเพิ่มประสิทธิภาพการใช้พลังงาน

การพิจารณาผลกระทบของงานเบื้องหลังต่อพลังงานเป็นสิ่งสำคัญในการสร้างแอป Android คุณภาพสูง ซึ่งจะช่วยให้คุณสร้างแอปที่มีความยืดหยุ่นและประหยัดแบตเตอรี่มากขึ้นได้ด้วยการทำความเข้าใจวิธีที่ระบบจัดการทรัพยากร การเพิ่มประสิทธิภาพเพื่อประหยัดพลังงานจะช่วยลดการใช้แบตเตอรี่ ป้องกันไม่ให้ระบบปิดแอป และมอบประสบการณ์การใช้งานที่ราบรื่นยิ่งขึ้น

ทำความเข้าใจขีดจำกัดของทรัพยากรการจัดการพลังงาน

ระบบจะจัดลําดับความสําคัญของคําขอทรัพยากรของแอปตามสถานะอุปกรณ์ สถานะแอป และถังพักสแตนด์บายของแอป

ดูข้อมูลเพิ่มเติมเกี่ยวกับวิธีที่แอปของคุณสามารถทำงานภายในขีดจำกัดของแหล่งข้อมูลการจัดการพลังงานของอุปกรณ์ เพื่อให้ทำงานในเบื้องหลังได้อย่างน่าเชื่อถือ
