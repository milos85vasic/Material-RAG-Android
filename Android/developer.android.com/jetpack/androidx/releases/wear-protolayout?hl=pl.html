wear protolayout

  
Dokumentacja API
androidx.wear.protolayout
Ta biblioteka umożliwia zdefiniowanie zestawu układów interfejsu użytkownika i wyrażeń niebędących interfejsem użytkownika, które mają być renderowane i ocenione na zdalnych powierzchniach.
Ostatnia aktualizacja Wersja stabilna Wersja kandydująca do publikacji Wersja Beta Wersja alfa
4 czerwca 2025 r. 1.3.0 - - -

Deklarowanie zależności

Aby dodać zależność od pakietu wear-protolayout, musisz dodać repozytorium Google Maven do projektu. Więcej informacji znajdziesz w repozytorium Maven firmy Google.

Dodaj zależności dla artefaktów, których potrzebujesz, w pliku build.gradle aplikacji lub modułu:

Odlotowe

dependencies {
    // Use to implement support for Wear ProtoLayout Expressions
    implementation "androidx.wear.protolayout:protolayout-expression:1.3.0"

    // Use to implement support for Wear ProtoLayout
    implementation "androidx.wear.protolayout:protolayout:1.3.0"

    // Use to utilize components and layouts with Material design in your ProtoLayout
    implementation "androidx.wear.protolayout:protolayout-material:1.3.0"
}

Kotlin

dependencies {
    // Use to implement support for Wear ProtoLayout Expressions
    implementation("androidx.wear.protolayout:protolayout-expression:1.3.0")

    // Use to implement support for Wear ProtoLayout
    implementation("androidx.wear.protolayout:protolayout:1.3.0")

    // Use to utilize components and layouts with Material design in your ProtoLayout
    implementation("androidx.wear.protolayout:protolayout-material:1.3.0")
}

Więcej informacji o zależnościach znajdziesz w artykule Dodawanie zależności kompilacji.

Opinia

Twoja opinia pomoże nam ulepszyć Jetpacka. Jeśli zauważysz nowe problemy lub masz pomysły na ulepszenie tej biblioteki, daj nam znać. Zanim utworzysz nowy problem, zapoznaj się z dotychczasowymi problemami w tej bibliotece. Możesz dodać swój głos do istniejącego problemu, klikając przycisk z gwiazdką.

Wersja 1.3.0

4 czerwca 2025 r.

Element androidx.wear.protolayout:protolayout-*:1.3.0 został zwolniony. Wersja 1.3.0 zawiera te komity.

Ważne zmiany od wersji 1.2.0

  • Zegarek wykorzystuje interfejs Material 3 z elementami i układami zoptymalizowanymi pod kątem okrągłego ekranu oraz skalowanymi odpowiednio do małych i dużych ekranów.
    • Obejmuje to bibliotekę protolayout-material3 przeznaczoną tylko na Kotlina, która zawiera więcej interfejsów API podobnych do Compose do obsługi tych komponentów i funkcji:
    • dynamiczny motyw kolorystyczny pochodzący z systemu i tarczy zegarka z najnowszym motywem Material3 dotyczącym kolorów, kształtów i typografii;
    • MaterialScope za zadbanie o wszystkie domyślne wartości i łatwiejszą personalizację
    • iconEdgeButton, textEdgeButton
    • iconButton, textButton, button, imageButton, avatarButton, compactButton
    • titleCard, appCard, graphicDataCard, iconDataCard, textDataCard
    • circularProgressIndicator, segmentedCircularProgressIndicator
    • primaryLayout, buttonGroup
    • Wszystkie komponenty działają na wszystkich poziomach pakietu SDK i w przypadku wersji ProtoLayout Renderera, zapewniając odpowiednie rozwiązania zastępcze w odpowiednich przypadkach.
  • Więcej interfejsów API podobnych do interfejsów Compose, które są przyjazne dla Kotlina, dla podstawowych elementów ProtoLayout.
    • LayoutModifier z możliwością dodawania większości modyfikatorów jako połączonych funkcji (padding, contentDescription (w tym clearSemantics), background, clip, opacity itp.), które można przekształcić w istniejący obiekt Modifiers
    • LayoutColors i LayoutString jako typy z łatwiejszym wsparciem dla pól i ograniczeń dynamicznych
    • text i fontStyle
    • lepsza obsługa mapy w przypadku StateBuilder, w tym DynamicDataMap i metod fabrycznych, takich jak intAppDataKey, ułatwiających tworzenie obiektów AppDataKey
  • Obsługa animacji Lottie, w tym opcja ustawiania różnych wyzwalaczy uruchamiania animacji, np. ładowania układu lub jego widoczności.
    • Ulepszona obsługa gradientów:
    • Interfejs API gradientu liniowego jako część interfejsu Brush, który może być używany w modyfikatorach Background w przypadku elementów takich jak Box, Spacer itp.
    • Zezwalanie na dynamiczne wartości kolorów w elementach ColorStop używanych do gradientu liniowego i gradientu przesunięcia
    • Dotychczasowy gradient zakrzywienia w obiektach łukowych obsługuje teraz dynamiczne kolory oraz kąty początkowy i końcowy.
  • Powiązanie danych platformy w protolayout-expression, aby otrzymywać informacje za każdym razem, gdy zmienia się widoczność układu. Może to na przykład służyć do ukrywania niektórych części układu podczas przesuwania kafelka.
  • Dodano bibliotekę testów (protolayout-testing), aby ułatwić testowanie jednostkowe wszystkich elementów prototypu.
  • Nowy element DashedArcLine z ulepszonym zestawem funkcji, dzięki któremu linia może zawierać kreski, w przeciwieństwie do dotychczasowego elementu ArcLine
  • ArcSpacer obsługa ustawiania długości w wymiarze DP zamiast w stopniach
  • Dodano oś zaokrąglenia FontSetting, która jest dostępna w przypadku niektórych czcionek

androidx.wear.protolayout:protolayout-*:1.3.0-rc01 nie zawiera żadnych zmian w porównaniu z poprzednią wersją. Wersja 1.3.0-rc01 zawiera te zatwierdzenia.

Wersja 1.3.0-beta02

7 maja 2025 r.

Element androidx.wear.protolayout:protolayout-*:1.3.0-beta02 został zwolniony. Wersja 1.3.0-beta02 zawiera te zatwierdzenia.

Poprawki błędów
