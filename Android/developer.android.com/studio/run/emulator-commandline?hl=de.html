Emulator über Befehlszeile starten
Mit Sammlungen den Überblick behalten Sie können Inhalte basierend auf Ihren Einstellungen speichern und kategorisieren.

Das Android SDK enthält einen Android-Geräteemulator, also ein virtuelles Gerät. das auf Ihrem Computer ausgeführt wird. Mit dem Android-Emulator können Sie Android-Apps ohne physisches Gerät

Auf dieser Seite werden Befehlszeilenfunktionen beschrieben, die Sie mit dem Android-Emulator Informationen zur Verwendung der Benutzeroberfläche des Android-Emulators finden Sie unter Apps im Android-Emulator ausführen

Emulator starten

Verwenden Sie den Befehl emulator, um den Emulator zu starten. Ihr Projekt auszuführen oder über den AVD Manager starten.

Hier sehen Sie die grundlegende Befehlszeilensyntax zum Starten eines virtuellen Geräts von einem Terminal-Eingabeaufforderung: 

emulator -avd avd_name [ {-option [value]} … ]

Oder 

emulator @avd_name [ {-option [value]} … ]

Wenn Sie den Emulator beispielsweise in Android Studio auf einem Mac starten, wird die Standardeinstellung sieht die Befehlszeile in etwa so aus: 

/Users/janedoe/Library/Android/sdk/emulator/emulator -avd Pixel8_API_34 -netdelay none -netspeed full -qt-hide-window -grpc-use-token -idle-grpc-timeout

Die Argumente -qt-hide-window -grpc-use-token -idle-grpc-timeout werden nur verwendet, um das Emulatorfenster in Android Studio auszuführen. Wenn Sie den Emulator in einem eigenen Fenster ausführen möchten, sollten Sie diese zusätzlichen Parameter nicht verwenden.

Sie können Startoptionen beim Starten des Emulators festlegen, aber nicht nach dem Start.

Geben Sie den folgenden Befehl ein, um eine Liste der AVD-Namen aufzurufen: 

emulator -list-avds

Mit dieser Option können Sie eine Liste der AVD-Namen aus Ihrem Android-Basisverzeichnis aufrufen. Sie können das Standard-Basisverzeichnis überschreiben, indem Sie die ANDROID_SDK_HOME festlegen die den Stamm des Objekts nutzerspezifisches Verzeichnis, in dem alle Konfigurations- und AVD-Inhalte gespeichert sind.

Sie können die Umgebungsvariable im Terminalfenster festlegen, bevor Sie eine virtuelles Gerät oder über die Nutzereinstellungen im System. Zum Beispiel in der Datei .bashrc unter Linux.

Schließen Sie das Emulatorfenster, um den Android-Emulator zu beenden.

App installieren

Neben der Installation einer App über Android Studio oder den Emulator UI haben, können Sie Ihre App mit dem adb-Dienstprogramm auf einem virtuellen Gerät installieren.

So verwenden Sie adb, um Ihre App zu installieren, auszuführen und zu testen: führen Sie die folgenden allgemeinen Schritte aus:

  1. Erstellen Sie Ihre App und verpacken Sie sie in einem APK, wie unter Build erstellen und ausführen App.
  2. Starten Sie den Emulator wie oben beschrieben über die Befehlszeile unter Verwendung der erforderlichen Startoptionen.
  3. Installieren Sie Ihre App mit adb.
  4. Führen Sie Ihre App im Emulator aus und testen Sie sie.
    Während der Emulator ausgeführt wird, können Sie den Emulator Konsole, um bei Bedarf Befehle auszuführen.

Gehen Sie bei der Deinstallation einer App wie bei einem Android-Gerät vor.

Das virtuelle Gerät behält die App und ihre Statusdaten bei Neustarts. in einer Laufwerkspartition für Nutzerdaten (userdata-qemu.img) Löschen starten Sie den Emulator mit der Option -wipe-data oder löschen Sie die Daten im AVD-Manager. Weitere Informationen zu den Nutzerdaten Partition und anderen Speicher finden Sie im folgenden Abschnitt.

Hinweis:Das Dienstprogramm adb betrachtet das virtuelle Gerät als physisches Gerät. Aus diesem Grund müssen Sie möglicherweise die Methode Das Flag -d mit einigen gängigen adb-Befehlen, z. B. install Mit dem Flag -d können Sie angeben, welches von mehreren verbundener Geräte, die als Ziel für einen Befehl verwendet werden sollen. Wenn Sie keine -d hat der Emulator ein Targeting auf das erste Gerät in der Liste.

Standardverzeichnisse und -dateien

Der Emulator verwendet verknüpfte Dateien, von denen das AVD-System und die Datenverzeichnisse sind am wichtigsten. Es ist hilfreich, die Struktur des Emulator-Verzeichnisses wenn Sie Befehlszeilenoptionen angeben, obwohl Sie normalerweise keine die Standardverzeichnisse oder -dateien ändern.

Der Android-Emulator verwendet den Hypervisor (QEMU).

AVD-Systemverzeichnis

Das Systemverzeichnis enthält die vom Emulator verwendeten Android-System-Images um das Betriebssystem zu simulieren. Dieses Verzeichnis enthält plattformspezifische, schreibgeschützte Dateien für alle AVDs desselben Typs, einschließlich API-Ebene, CPU-Architektur und Android-Variante. Die Standardspeicherorte sind:

  • macOS und Linux – ~/Library/Android/sdk/system-images/android-apiLevel/variant/arch/
  • Windows – C:\Users\user\Library\Android\sdk\system-images\android-apiLevel\variant\arch\

Dabei gilt:

  • apiLevel ist eine numerische API-Ebene oder ein Buchstabe für Vorabversionen. Beispielsweise gibt android-V die Android- Vorschau von VanillaIceCream. Nach der Veröffentlichung wurde es API-Level 35, das von android-35
  • variant ist ein Name, der bestimmten Elementen entspricht. System-Image implementiert wird. Beispiel: google_apis oder android-wear.
  • arch ist die Ziel-CPU-Architektur. Beispiel: x86

Mit der Option -sysdir können Sie ein anderes Systemverzeichnis für der AVD.

AVD-Datenverzeichnis

Das AVD-Datenverzeichnis, auch Inhaltsverzeichnis genannt, ist spezifisch für nur eine AVD-Instanz erstellen und alle änderbaren Daten für das AVD enthalten.

Der Standardspeicherort ist der folgende, wobei name der Wert für AVD-Name:

  • macOS und Linux – ~/.android/avd/name.avd/
  • Windows 10 und höher – C:\Users\user\.android\name.avd\

Mit der Option -datadir können Sie ein anderes AVD-Datenverzeichnis angeben.

emulator @Pixel8_API_34 -initdata

emulator @Pixel8_API_34 -kernel

emulator @Pixel8_API_34 -noaudio

emulator @Pixel8_API_34 -no-snapshot

Hilfe zu Befehlszeilenoptionen

In diesem Abschnitt wird beschrieben, wie Sie Hilfe zu den Befehlszeilenoptionen erhalten. Es ausführlichere Informationen zu häufig verwendeten Befehlszeilenoptionen für den Emulator, die beim Starten des Emulators verfügbar sind.

Alle Emulator-Optionen auflisten

Um eine Liste aller Emulatoroptionen einschließlich einer kurzen Beschreibung zu drucken, geben Sie folgenden Befehl: 

emulator -help

Detaillierte Hilfe zu einer bestimmten Option erhalten

Geben Sie den folgenden Befehl ein, um Hilfe für eine bestimmte Startoption zu drucken: 

emulator -help-option

Beispiel: 

emulator -help-netspeed

Diese Hilfe ist ausführlicher als die Beschreibung des Option -help.

Detaillierte Hilfe für alle Optionen

Geben Sie den folgenden Befehl ein, um detaillierte Hilfe zu allen Emulatoroptionen zu erhalten: 

emulator -help-all

Umgebungsvariablen des Emulators auflisten

Geben Sie den folgenden Befehl ein, um eine Liste der Umgebungsvariablen des Emulators aufzurufen: 

emulator -help-environment

Sie können Umgebungsvariablen im Terminalfenster festlegen, bevor Sie eine virtuelles Gerät oder Sie legen dies in den Nutzereinstellungen im System. Du kannst sie beispielsweise unter Linux in der Datei .bashrc festlegen.

Debug-Tags auflisten

Geben Sie den folgenden Befehl ein, um eine Liste der Tags für die -debug-Optionen auszugeben: 

emulator -help-debug-tags

Mit den Optionen für -debug kannst du Debug-Meldungen für folgende Elemente aktivieren oder deaktivieren: Emulatorkomponenten angeben, wie in den Tags angegeben.
