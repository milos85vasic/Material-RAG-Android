API 參考資料
androidx.wear.protolayout
這個程式庫可定義要在遠端介面上轉譯/評估的一組 UI 版面配置和非 UI 運算式。
最近更新時間 穩定版 候選版 Beta 版 Alpha 版
2025 年 6 月 4 日 1.3.0 - - -

宣告依附元件

如要為 wear-protolayout 新增依附元件，您必須將 Google Maven 存放區新增至專案。詳情請參閱 Google 的 Maven 存放區。

在應用程式或模組的 build.gradle 檔案中，新增所需構件的依附元件：

Groovy

dependencies {
    // Use to implement support for Wear ProtoLayout Expressions
    implementation "androidx.wear.protolayout:protolayout-expression:1.3.0"

    // Use to implement support for Wear ProtoLayout
    implementation "androidx.wear.protolayout:protolayout:1.3.0"

    // Use to utilize components and layouts with Material design in your ProtoLayout
    implementation "androidx.wear.protolayout:protolayout-material:1.3.0"
}

Kotlin

dependencies {
    // Use to implement support for Wear ProtoLayout Expressions
    implementation("androidx.wear.protolayout:protolayout-expression:1.3.0")

    // Use to implement support for Wear ProtoLayout
    implementation("androidx.wear.protolayout:protolayout:1.3.0")

    // Use to utilize components and layouts with Material design in your ProtoLayout
    implementation("androidx.wear.protolayout:protolayout-material:1.3.0")
}

如要進一步瞭解依附元件，請參閱「新增建構依附元件」一文。

1.3.0 版本

2025 年 6 月 4 日

發布 androidx.wear.protolayout:protolayout-*:1.3.0。1.3.0 版包含這些修訂版本。

自 1.2.0 版以來的重要異動

  • 手錶採用 Material 3 設計，其中的元件和版面配置經過最佳化，可配合圓形螢幕顯示，並適當調整從小螢幕到大螢幕的大小
    • 這包括僅限 Kotlin 的 protolayout-material3 程式庫，其中包含更多類似 Compose 的 API，可用於下列元件和功能：
    • 系統和錶面提供的動態色彩主題，採用最新的 Material3 主題，可設定顏色、形狀和字體排版
    • MaterialScope 可處理所有自訂預設值，並簡化自訂程序
    • iconEdgeButton、textEdgeButton
    • iconButton、textButton、button、imageButton、avatarButton、compactButton
    • titleCard、appCard、graphicDataCard、iconDataCard、textDataCard
    • circularProgressIndicator、segmentedCircularProgressIndicator
    • primaryLayout、buttonGroup
    • 所有元件都能在任何 SDK 級別和 ProtoLayout 轉譯器版本中運作，並在適用情況下提供合理的備用方案
  • 適用於基本 ProtoLayout 元素的 Kotlin 友好型 Compose 類似 API
    • LayoutModifier 可將大部分修飾符設為連接的函式 (padding、contentDescription (包括 clearSemantics)、background、clip、opacity 等)，可轉換為現有的 Modifiers 物件
    • LayoutColors 和 LayoutString 可做為類型，更輕鬆地支援使用動態欄位和限制
    • text 和 fontStyle
    • 為 StateBuilder 提供更完善的對應功能，包括 DynamicDataMap 和 intAppDataKey 等工廠方法，以便輕鬆建立 AppDataKey 物件
  • Lottie 動畫支援的選項包括設定動畫開始播放時的不同觸發事件，例如版面配置載入或顯示時
    • 改善漸層支援功能：
    • 線性漸層 API 是 Brush 的一部分，可用於 Background 修飾符，用於 Box、Spacer 等元素。
    • 允許在 ColorStop 中使用動態顏色值，用於線性和掃描漸層
    • 弧形物件中現有的掃描漸層現在支援動態顏色和起始/結束角度
  • 在 protolayout-expression 中繫結平台資料，以便在版面配置的顯示設定變更時隨時接收資訊，例如在輕掃資訊方塊時隱藏版面配置的特定部分。
  • 新增測試程式庫 - protolayout-testing，以便為任何 ProtoLayout 元素提供更簡單的單元測試涵蓋率
  • 新增 DashedArcLine 元素，提供更完善的功能組合，讓線條可以包含破折號，而非現有的 ArcLine
  • ArcSpacer 支援以 DP 維度而非度數設定長度
  • 新增適用於部分字型的圓滑度 FontSetting 軸

1.3.0-beta02 版

2025 年 5 月 7 日

發布 androidx.wear.protolayout:protolayout-*:1.3.0-beta02。1.3.0-beta02 版包含這些修訂項目。

修正錯誤

  • 我們已針對適用於 API 36 以上版本的字體排版設計值，做出重大改善。這是因為從 API 36 開始，所有資訊方塊都會採用系統字型，因此這項變更可讓資訊方塊輪轉介面更一致。(If316f)。
  • 如果未設定動態值的 layoutConstraints，Text、Spacer、ArcLine 和 DashedArcLine 建構工具就不會擲回。請注意，舊版轉譯器仍需要設定 layoutConstraints，並會忽略未設定 layoutConstraints 的任何動態值。(Ic52e8)。
  • 新增標題語義修飾符，指出版面配置元素是無障礙目的的內容區段標題，並預設標示 primaryLayout 標題區塊中的文字為無障礙標題。(Iae1fb)
  • primaryLayout 的最終使用者體驗優化：在較小螢幕上，標題區塊與主區塊之間的間距從 6dp 縮減為 4dp。(I0e056)
  • 將預設內容說明套用至 textEdgeButton。(Ifaf8b)。
  • 微幅更新 avatarButton 內部邊框間距。(I0910b)。

1.3.0-beta01 版本

2025 年 4 月 9 日

發布 androidx.wear.protolayout:protolayout-*:1.3.0-beta01。1.3.0-beta01 版包含這些修訂版本。

新功能

Wear ProtoLayout 的 1.3.0-beta01 版本指出，此程式庫版本功能完整且 API 已鎖定 (標示為實驗功能的除外)。Wear ProtoLayout 1.3 包含下列新功能和 API：

  • 手錶採用 Material 3 設計，其中的元件和版面配置經過最佳化，可配合圓形螢幕顯示，並適當調整從小螢幕到大螢幕的大小
    • 這包括僅限 Kotlin 的 protolayout-material3 程式庫，其中包含更多類似 Compose 的 API，可用於下列元件和功能：
    • 系統和錶面提供的動態色彩主題，採用最新的 Material3 主題，可設定顏色、形狀和字體排版
    • MaterialScope 可處理所有自訂預設值，並簡化自訂程序
    • iconEdgeButton、textEdgeButton
    • iconButton、textButton、button、imageButton、avatarButton、compactButton
    • titleCard、appCard、graphicDataCard、iconDataCard、textDataCard
    • circularProgressIndicator、segmentedCircularProgressIndicator
    • primaryLayout、buttonGroup
    • 所有元件都能在任何 SDK 級別和 ProtoLayout 轉譯器版本中運作，並在適用情況下提供合理的備用方案
  • 適用於基本 ProtoLayout 元素的 Kotlin 友好型 Compose 類似 API
    • LayoutModifier 可將大部分修飾符設為連接的函式 (padding、contentDescription (包括 clearSemantics)、background、clip、opacity 等)，可轉換為現有的 Modifiers 物件
    • LayoutColors 和 LayoutString 可做為類型，更輕鬆地支援使用動態欄位和限制
    • text 和 fontStyle
    • 為 StateBuilder 提供更完善的對應功能，包括 DynamicDataMap 和 intAppDataKey 等工廠方法，以便輕鬆建立 AppDataKey 物件
  • Lottie 動畫支援的選項包括設定動畫開始播放時的不同觸發事件，例如版面配置載入或顯示時
    • 改善漸層支援功能：
    • 線性漸層 API 是 Brush 的一部分，可用於 Background 修飾符，用於 Box、Spacer 等元素。
    • 允許在 ColorStop 中使用動態顏色值，用於線性和掃描漸層
    • 弧形物件中現有的掃描漸層現在支援動態顏色和起始/結束角度
  • 在 protolayout-expression 中繫結平台資料，以便在版面配置的顯示設定變更時隨時接收資訊，例如在輕掃資訊方塊時隱藏版面配置的特定部分。
  • 新增測試程式庫 - protolayout-testing，以便為任何 ProtoLayout 元素提供更簡單的單元測試涵蓋率
  • 新增 DashedArcLine 元素，提供更完善的功能組合，讓線條可以包含破折號，而非現有的 ArcLine
  • ArcSpacer 支援以 DP 維度而非度數設定長度
  • 新增適用於部分字型的圓滑度 FontSetting 軸

1.2.0-beta01 版本

2024 年 7 月 10 日

發布 androidx.wear.protolayout:protolayout-*:1.2.0-beta01。1.2.0-beta01 版包含這些修訂版本。

新功能

Wear ProtoLayout 的 1.2.0-beta01 版本指出，此程式庫版本功能完整且 API 已鎖定 (標示為實驗性功能的除外)。Wear ProtoLayout 1.2 包含下列新功能和 API：

  • FontStyle 已更新為支援以下額外的字型：
    • 設定不同的字型變化設定，例如 FontSetting.weight 和 FontSetting.width
    • 為所有數字字元設定相同的寬度 - 表格式數字 (FontSetting.tnum 字型功能設定)
    • 改善字型選取 API，透過指定要使用的偏好字型系列名稱，支援即將推出的 Flex 字型。
  • 擴充 Corner 修飾符，以便使用不同的水平和垂直值指定每個 CornerRadius，讓您建立不對稱角落的元素。
  • 新增 Transformation 修飾符，提供 LayoutElement 的平移、旋轉和縮放功能。您可以使用動態值為這些轉換效果製作動畫。
  • 在所有弧形元素 (Arc、ArcLine 和 ArcText) 中新增 setArcDirection 和 Clockwise、CounterClockwise 和 Normal 選項，以便在不同版面配置方向 (例如 LTR 和 RTL) 中提供更完善的支援。
  • 透過擴大使用 Clickable 修飾符的任何元素可點選區域，使其至少為 48dp 乘以 48dp，改善所有觸控目標的無障礙性。
  • 新增 setResponsiveContentInsetEnabled 來改善 PrimaryLayout 和 EdgeContentLayout，以便在不同螢幕大小下，更妥善地支援這些版面配置的回應式行為，並改善方塊的一致性。新增 Linter 警告，建議使用這些 API 並提供快速修正方式。
  • 改善 Material Text 的縮放/非縮放功能，以便在 Android 14 中使用非線性字型縮放功能。

API 變更

  • 預設字型系列名稱 (DEFAULT_SYSTEM_FONT) 已移除，因為不使用 preferredFontFamilies API 會隱含這項資訊。(I39dab)
  • 傳遞至 FontSetting.width 的參數應為正值。(I1266f)。

  • 支援在 Material Text 中使用 StringProp 搭配 DynamicString。此功能適用於支援 1.2 版的轉譯器。舊版轉譯器會改用提供的靜態值。將 Text#getText 傳回類型從 String 更新為 StringProp。(I7275b)
