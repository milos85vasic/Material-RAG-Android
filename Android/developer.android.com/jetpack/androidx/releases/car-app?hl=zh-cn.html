声明依赖项

如需添加汽车应用库的依赖项，您必须将 Google Maven 制品库添加到项目中。如需了解详情，请参阅 Google 的 Maven 制品库。

在应用或模块的 build.gradle 文件中添加所需工件的依赖项：

Groovy

dependencies {
    implementation "androidx.car.app:app:1.7.0"

    // For Android Auto specific functionality
    implementation "androidx.car.app:app-projected:1.7.0"

    // For Android Automotive specific functionality
    implementation "androidx.car.app:app-automotive:1.7.0"

    // For testing
    testImplementation "androidx.car.app:app-testing:1.7.0"
}

Kotlin

dependencies {
    implementation("androidx.car.app:app:1.7.0")

    // For Android Auto specific functionality
    implementation("androidx.car.app:app-projected:1.7.0")

    // For Android Automotive specific functionality
    implementation("androidx.car.app:app-automotive:1.7.0")

    // For testing
    testImplementation("androidx.car.app:app-testing:1.7.0")
}

如需详细了解依赖项，请参阅添加 build 依赖项。

版本 1.8

版本 1.8.0-alpha02

2025 年 6 月 18 日

发布了 androidx.car.app:app-*:1.8.0-alpha02。版本 1.8.0-alpha02 中包含这些提交内容。

新功能

  • 添加了对 Action.MEDIA_PLAYBACK 的支持，该功能可根据媒体播放状态显示指示器。
  • 添加了对在刷新 SectionedItemTemplate 时控制用户滚动位置的支持。

API 变更

  • 添加了对在刷新之间保存用户在 SectionedItemTemplate 中的滚动位置的支持。(Ia4c51)
  • 更新了 MEDIA_PLAYBACK 操作，以针对行结束操作 (I05cc4) 启用该操作
  • 添加了新的操作类型 (I6cc5a)
  • 添加 CarAppExtender#addAction(Action)，以便将任何类型的操作添加到 CarAppExtender。(Idc4d7)
  • 使 SectionedItemTemplate 可用于 API 8 及更高版本。(I9a079)

版本 1.8.0-alpha01

2025 年 5 月 20 日

发布了 androidx.car.app:app-*:1.8.0-alpha01。版本 1.8.0-alpha01 中包含这些提交内容。

新功能

所有功能暂时仅适用于 Android Auto。在整个 AAOS 中使用相同的 API

  • 添加了媒体类别，用于创建自定义应用，作为基于 MediaBrowse 的应用的替代方案
  • 添加了播放模板，用于在媒体播放视图中控制顶部操作（仅适用于媒体应用）
  • 在 Android Auto 上全面支持 Sectioned Item Template，从而允许组合由列表/网格组成的子部分。
  • 为网格项添加了超大尺寸。

API 变更

  • 为网格项添加了额外的尺寸选项 (I35b58)
  • 为媒体应用向汽车应用库添加了类别和权限 (I8e100)
  • 添加了新 API，供应用检测是否支持 CarAppLibrary 媒体类别 (Ic4b08)
  • 已将 OnItemVisibilityChangeListener 添加到部分。(I2c2fd)
  • 添加了 Mileage#getOdometerKilometers 并弃用了 Mileage#getOdometerMeters (Ic91af)

bug 修复

  • 修复了 CarIcon.setTint 的 Javadoc 中的拼写错误 (Iabd72)
  • 更新了 Row.Builder#setNumericDecoration 文档，以反映不支持 1 个操作 + 数字装饰；但支持 0 个或 2 个操作 + 数字装饰。(Ic0b08)
  • 为网格项添加了徽章图标背景颜色字段。(I2b6ae)

版本 1.7.0-beta03

2024 年 11 月 13 日

修复了安全漏洞和其他常规 bug。如果您使用的是较低版本，请更新以使用此版本。

发布了 androidx.car.app:app-*:1.7.0-beta03。版本 1.7.0-beta03 中包含这些提交内容。

API 变更

  • 更新了 CAL 序列化 / 反序列化代码，使其仅处理声明了 @CarProtocol 注释的对象。(Ic730e)
  • 现在，CarAppExtender 可用于扩展框架 Notification.Builder，而不仅仅是 NotificationCompat.Builder。(Id3ad7)
  • 添加了 KEY_EXCLUDE_MEDIA_ITEM_FROM_MIXED_APP_LIST extra。(I201f9)
  • 弃用了 ConversationItem 上空的 Builder 构造函数，并将其替换为接受必需参数的构造函数。还添加了一项检查，以防出现 null 消息。(Ic8221)

bug 修复

  • 确保 PlaceList 地图使用字符串（纯文本）标题。(Ic992f)
  • 此库现在使用类型使用的 JSpecify nullness 注释。Kotlin 开发者应使用以下编译器实参来强制执行正确用法：-Xjspecify-annotations=strict（这是 Kotlin 编译器从 2.1.0 版开始的默认实参）。（Ib5367、b/326456246）

版本 1.7.0-beta02

2024 年 9 月 18 日

发布了 androidx.car.app:app-*:1.7.0-beta02。版本 1.7.0-beta02 中包含这些提交内容。

API 变更

  • 添加了 KEY_ROOT_HINT_MEDIA_HOST_VERSION 额外内容 (I8796b)
  • 将 TabContents.Builder 类中的 TabContents.Api8Builder 类替换为 @ExperimentalCarApi 构造函数 (I26fbe)
  • 为 CarMediaApp 添加了 intent 操作和 extra (I50782)
  • 将消息传递 API 标记为非实验性 API (I0b070)
  • 向 SectionedItemTemplate 添加了远程项加载功能（允许加载长列表而不会崩溃）(I0d122)
  • 在 API 8 中，将 SectionedItemTemplate 添加到 TabTemplate 内的支持的模板列表中。(Idc5d6)
