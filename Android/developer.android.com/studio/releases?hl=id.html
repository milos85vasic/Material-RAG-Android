Update Fitur Baru Android Studio Narwhal 3 | 2025.1.3
Tetap teratur dengan koleksi Simpan dan kategorikan konten berdasarkan preferensi Anda.

Android Studio adalah IDE resmi untuk pengembangan Android, dan dilengkapi semua hal yang Anda perlukan untuk membangun aplikasi Android.

Halaman ini mencantumkan fitur baru dan peningkatan dalam versi terbaru di saluran stabil, Update Fitur Baru 3 Android Studio Narwhal. Anda dapat mendownloadnya di sini atau meng-updatenya di dalam Android Studio dengan mengklik Help > Check for updates (Android Studio > Check for updates di macOS)

Ini adalah rilis stabil Android Studio. Perhatikan bahwa patch mungkin berisi fitur minor baru dan perbaikan bug. Lihat Nama rilis Android Studio untuk memahami penamaan versi Android Studio.

Untuk melihat apa yang telah diperbaiki dalam versi Android Studio ini, lihat masalah yang ditutup.

Untuk melihat catatan rilis Android Studio versi lama, lihat Rilis sebelumnya.

Untuk akses awal ke fitur dan peningkatan mendatang, lihat build Pratinjau Android Studio.

Jika Anda mengalami masalah saat menggunakan Android Studio, lihat halaman Masalah umum atau Pemecahan masalah.

Kompatibilitas Android Studio dan plugin Android Gradle

Sistem build Android Studio didasarkan pada Gradle. Plugin Android Gradle (AGP) menambahkan beberapa fitur yang dikhususkan untuk membangun aplikasi Android. Tabel berikut mencantumkan versi AGP yang diperlukan untuk setiap versi Android Studio.

Sekarang ada kebijakan kompatibilitas berbasis waktu untuk AGP dan Android Studio. Setiap versi Android Studio akan mendukung versi AGP yang dirilis dalam 3 tahun sebelumnya. Versi AGP yang lebih lama dari 3 tahun tidak akan didukung lagi di Android Studio versi yang lebih baru. Jika versi AGP project Anda tidak kompatibel, Android Studio akan meminta Anda untuk mengupdate AGP.
Jika project Anda tidak didukung oleh versi Android Studio tertentu, Anda tetap dapat membuka dan mengupdate project menggunakan Android Studio versi lama.

Kompatibilitas Android Studio dan layanan Cloud

Android Studio menyertakan integrasi layanan yang membantu Anda dan tim membuat kemajuan lebih cepat saat mengembangkan, merilis, dan memelihara aplikasi Android. Hal ini mencakup layanan Cloud seperti Gemini di Android Studio, Play Vitals, dan Firebase Crashlytics. Layanan cloud hanya tersedia di versi saluran stabil terbaru Android Studio dan versi utama (termasuk patch-nya) yang dirilis dalam 10 bulan sebelumnya. Jika versi berada di luar rentang kompatibilitas tersebut, integrasi layanan akan dinonaktifkan dan Anda harus mengupdate Android Studio.

Kebijakan kompatibilitas layanan Cloud ini diumumkan dengan rilis saluran stabil Update Fitur Baru Android Studio Meerkat. Penerapan akan dimulai dengan rilis saluran stabil Update Fitur Baru Android Studio Narwhal, yang akan keluar dari jendela kompatibilitas pada Kuartal 2 tahun 2026.

Versi Android Studio yang saat ini kompatibel dengan layanan Cloud

Mulai dari rilis saluran stabil Update Fitur Baru Android Studio Meerkat, semua versi Studio kompatibel dengan layanan Cloud. Penerapan akan dimulai dengan Update Fitur Baru Narwhal Android Studio.

Insight Kebijakan Google Play di Android Studio

Android Studio kini menyertakan insight dan panduan yang lebih lengkap tentang kebijakan Google Play yang dapat memengaruhi aplikasi Anda. Informasi ini membantu Anda membangun aplikasi yang lebih aman sejak awal, sehingga mencegah masalah yang dapat mengganggu proses peluncuran dan menghabiskan lebih banyak waktu dan sumber daya untuk diperbaiki nanti.

Mulai dari Update Fitur Baru Android Studio Narwhal 3, Anda dapat melihat Insight Kebijakan Play sebagai pemeriksaan lint. Pemeriksaan lint ini akan menampilkan informasi berikut:

  • Ringkasan kebijakan.
  • Anjuran dan larangan untuk menghindari kesalahan umum.
  • Link ke halaman kebijakan Play tempat Anda dapat menemukan kebijakan formal lengkap dan informasi serta referensi bermanfaat lainnya.

Fitur ini dimaksudkan untuk memberikan panduan prapeninjauan yang bermanfaat sehingga Anda dapat memiliki pengalaman pengiriman aplikasi yang lebih lancar. Fitur ini tidak mencakup setiap kebijakan, dan tidak memberikan keputusan akhir peninjauan aplikasi. Selalu tinjau kebijakan selengkapnya di Pusat Kebijakan untuk mendapatkan panduan. Kami juga secara aktif mengembangkan dan meningkatkan kualitas integrasi ini. Jika Anda memiliki masukan, harap laporkan.

Untuk melihat apakah ada Insight Kebijakan Google Play untuk project Anda, buka Code > Inspect for Play Policy Insights… Insight akan tercantum di jendela alat 'Problems' dan juga akan muncul sebagai peringatan Lint di file yang sesuai.

Menguji dan mengembangkan dengan pencadangan dan pemulihan aplikasi

Memastikan Pencadangan dan Pemulihan Android berfungsi dengan benar untuk aplikasi Anda adalah aspek penting untuk memastikan pengguna tetap berinteraksi dengan aplikasi Anda setelah beralih ke perangkat baru atau memulihkan dari cloud. Namun, menguji apakah pencadangan dan pemulihan data berfungsi untuk aplikasi Anda bisa sulit.

Rilis Fitur Android Studio Narwhal 3 menyediakan cara bagi Anda untuk membuat cadangan aplikasi dan memulihkannya ke perangkat lain. Hal ini dapat berguna untuk menguji apakah aplikasi Anda berperilaku seperti yang diharapkan saat memulihkan data aplikasi dari perangkat ke perangkat atau dari cadangan cloud, atau jika Anda menginginkan cara yang lebih cepat untuk menyiapkan perangkat pengujian dengan data yang Anda butuhkan untuk mengembangkan dan men-debug aplikasi.

Membuat cadangan

Untuk membuat file cadangan, lakukan hal berikut:

  1. Men-deploy versi debug aplikasi Anda ke perangkat yang terhubung
  2. Gunakan salah satu tindakan berikut untuk membuat cadangan:
    • Dari jendela Running Device, klik tindakan Backup App Data dari toolbar
    • Pilih Run > Backup App Data dari panel menu utama
    • Dari tab Device Explorer > Processes, klik kanan proses aplikasi, lalu pilih Backup App Data
  3. Pada dialog yang muncul, lakukan hal berikut:
    • Konfirmasi ID aplikasi untuk aplikasi yang ingin Anda buat cadangannya
      • Pilih apakah Anda ingin membuat cadangan Perangkat ke Perangkat, Cloud, atau Cloud (Tidak Terenkripsi)
    • Konfirmasi nama dan lokasi cadangan yang ingin Anda simpan. Secara default, cadangan disimpan ke direktori root project Android Studio saat ini.
  4. Klik Oke saat Android Studio menanyakan apakah aplikasi dapat dihentikan. Untuk membuat cadangan, Android Studio harus menghentikan proses aplikasi.

Anda dapat melihat cadangan yang Anda buat di jendela alat Project > Android di bawah node Backup Files.

Jenis cadangan

Anda dapat membuat berbagai jenis cadangan untuk aplikasi Anda. Saat membuat cadangan, pilih jenis cadangan yang terkait dengan skenario yang ingin Anda uji:

  • Perangkat ke perangkat: Membuat cadangan aplikasi Anda, mirip dengan cadangan yang dibuat selama transfer perangkat ke perangkat. Dalam transfer perangkat ke perangkat, data pencadangan aplikasi dikirim langsung ke perangkat lain, misalnya melalui USB atau Wi-Fi.
  • Cloud: Membuat cadangan aplikasi Anda, mirip dengan cadangan yang disimpan ke penyimpanan Akun Google pengguna. Saat menyiapkan perangkat baru, pengguna dapat memilih untuk memulihkan dari cadangan Cloud.
  • Cloud (tidak dienkripsi): Membuat cadangan aplikasi Anda, serupa dengan cadangan yang disimpan ke penyimpanan Akun Google pengguna di perangkat yang tidak mengaktifkan enkripsi sisi klien.

Catatan: Saat menggunakan fitur untuk menguji Cloud, Cloud (Tidak Terenkripsi) atau Pencadangan Perangkat ke Perangkat, cadangan yang dibuat tidak dikirim ke Cloud atau ke perangkat lain, tidak seperti alur pencadangan sebenarnya. Layanan ini membuat cadangan aplikasi Anda seolah-olah akan disimpan ke Cloud atau dikirim ke perangkat lain. Cadangan yang dibuat untuk jenis Pencadangan apa pun dapat berada di bagian File Cadangan di jendela alat Project > Android.

Memulihkan data aplikasi

Untuk memulihkan data aplikasi, lakukan hal berikut:

  1. Deploy aplikasi Anda ke perangkat yang terhubung. Aplikasi harus memiliki ID aplikasi yang sama dengan file cadangan yang ingin Anda pulihkan ke perangkat.
  2. Buka dan klik salah satu tindakan berikut:
    • Dari jendela Running Device, klik tindakan Restore App Data dari toolbar, lalu pilih file cadangan dari histori terbaru atau klik Browse.
    • Buka Run > Restore App Data dari panel menu utama.
    • Dari tab Device Explorer > Processes, klik kanan proses aplikasi dan pilih Restore App Data.
    • Dari jendela alat Project > Android, klik kanan cadangan di bawah node Backup Files, lalu pilih Restore App Data.
  3. Jika berlaku, pilih cadangan dari histori terbaru atau klik Jelajahi untuk memilih file cadangan yang akan dipulihkan dari penyimpanan lokal.

Atau, Anda dapat menyertakan file cadangan sebagai bagian dari konfigurasi run, sehingga men-deploy aplikasi juga memulihkan data aplikasi dari file cadangan. Untuk melakukannya, lakukan hal berikut:

  1. Buka Run > Edit Configurations dari panel menu utama.
  2. Pilih konfigurasi eksekusi aplikasi dan buka Opsi pemulihan.
  3. Untuk memulihkan aplikasi dari cadangan, centang kotak di samping Pulihkan status aplikasi.
  4. Pilih file cadangan dari histori terbaru atau jelajahi dan pilih file cadangan dari penyimpanan lokal.
  5. Jika Anda hanya ingin memulihkan data aplikasi pada penginstalan aplikasi baru, centang kotak di samping Hanya pulihkan pada penginstalan APK baru. Opsi ini dapat berguna jika Anda men-deploy ke perangkat pengujian baru dan ingin memulihkan data untuk membantu proses penelusuran bug dan pengembangan aplikasi.
  6. Klik OK untuk menyimpan konfigurasi run.
  7. Deploy aplikasi Anda menggunakan konfigurasi run untuk menguji pemulihan data aplikasi Anda ke perangkat yang terhubung.

Pemeriksaan Proguard

Android Studio kini menyertakan pemeriksaan untuk mencegah aturan Proguard yang dibuat dengan buruk, atau aturan yang mencegah pengoptimalan R8. Aturan penyimpanan yang terlalu luas seperti -keep class **.* dan konfigurasi aturan Proguard konsumen seperti dontshrink dan -dontoptimize kini memicu peringatan di IDE Studio. Untuk membuat aturan keep yang baik yang memungkinkan pengecilan kode, cakup aturan ke paket tertentu dan nyatakan secara eksplisit apa yang ingin Anda pertahankan.

File AGENT.md untuk konteks tingkat project

Sekarang Anda dapat menyertakan file AGENT.md dalam project Anda. File ini adalah file Markdown yang memberikan petunjuk khusus project, aturan gaya coding, dan panduan lainnya kepada Gemini sebagai konteks.

Gemini otomatis menemukan dan menerapkan petunjuk dari file apa pun yang bernama AGENT.md di project Anda. Jika file AGENT.md tidak ada, Gemini akan mencari file GEMINI.md sebagai penggantinya.

Setelan baru untuk menonaktifkan Sinkronisasi Otomatis

Android Studio kini menawarkan setelan untuk beralih dari mode Sinkronisasi Otomatis default (misalnya, Sinkronisasi berjalan otomatis saat project dibuka) ke mode Sinkronisasi Manual baru dengan pengingat. Perilaku defaultnya tetap Sinkronisasi Otomatis. Untuk beralih ke Sinkronisasi Manual, buka File (Android Studio di macOS) > Settings > Build, Execution, Deployment > Build Tools dan tetapkan Project Sync mode ke Manual Sync with reminders sebagai Project Sync mode.

Catatan: Anda mungkin tidak melihat setelan jika membuka Setelan segera setelah memulai Android Studio dan sebelum membuka project. Setelan akan muncul setelah Anda membuka project.

Lampiran gambar di Gemini

Anda kini dapat melampirkan file gambar dan memberikan informasi tambahan beserta perintah Anda. Misalnya: Anda dapat melampirkan tiruan UI atau screenshot untuk memberi tahu konteks Gemini tentang tata letak aplikasi Anda. Oleh karena itu, Gemini dapat membuat kode Compose berdasarkan gambar yang diberikan, atau menjelaskan composable dan alur data screenshot UI. Untuk mempelajari lebih lanjut, lihat Melampirkan gambar ke kueri Anda.

@Konteks file di Gemini

Anda kini dapat melampirkan file project sebagai konteks dalam interaksi chat dengan Gemini di Android Studio. Dengan begitu, Anda dapat dengan cepat mereferensikan file dalam perintah Anda untuk Gemini. Di input chat Gemini, ketik @ untuk membuka menu penyelesaian file dan pilih file yang akan dilampirkan. Anda juga dapat mengklik drop-down Konteks untuk melihat file mana yang dilampirkan secara otomatis oleh Gemini. Cara ini memberi Anda lebih banyak kontrol atas konteks yang dikirim ke Gemini. Untuk mempelajari lebih lanjut, lihat Melampirkan file ke kueri Anda.
