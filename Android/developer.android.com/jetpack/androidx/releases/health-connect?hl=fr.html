Santé Connect

  
Documentation de référence de l'API
androidx.health.connect.client
Permet aux développeurs de lire ou d'écrire les informations de santé et de remise en forme d'un utilisateur.
Dernière mise à jour Version stable Version finale Version bêta Version alpha
30 juillet 2025 - 1.1.0-rc03 - 1.2.0-alpha01

Demander l'accès aux types de données

Pour nous aider à renforcer la sécurité et la confidentialité des utilisateurs, les développeurs qui intègrent Santé Connect doivent déclarer l'accès en lecture et/ou en écriture aux types de données que leurs applications utilisent. Les développeurs doivent inclure des cas d'utilisation valides pour les types de données qu'ils utilisent en fonction de l'objectif de l'application. Pour en savoir plus, consultez Fournir des informations dans le formulaire de déclaration des applications de santé et Autorisations de Santé Connect par Android.

Déclarer des dépendances

Pour ajouter une dépendance sur Health, vous devez ajouter le dépôt Maven de Google à votre projet. Pour en savoir plus, consultez la section Dépôt Maven de Google.

Ajoutez les dépendances correspondant aux artefacts dont vous avez besoin dans le fichier build.gradle de votre application ou de votre module :

Groovy

dependencies {
    // Use to implement health connects
    implementation "androidx.health.connect:connect-client:1.2.0-alpha01"
}

Kotlin

dependencies {
    // Use to implement health connects
    implementation("androidx.health.connect:connect-client:1.2.0-alpha01")
}

Pour en savoir plus sur les dépendances, consultez Ajouter des dépendances de compilation.

Commentaires

Vos commentaires nous aident à améliorer Jetpack. N'hésitez pas à nous contacter si vous découvrez de nouveaux problèmes ou si vous avez des idées pour améliorer cette bibliothèque. Veuillez consulter les problèmes existants de cette bibliothèque avant d'en signaler un nouveau. Vous pouvez ajouter votre vote à un problème existant en cliquant sur le bouton en forme d'étoile.

Signaler un nouveau problème

Pour en savoir plus, consultez la documentation sur l'outil Issue Tracker.

Nouvelles fonctionnalités

La bibliothèque de test Santé Connect simplifie la création de tests automatisés. Vous pouvez utiliser cette bibliothèque pour vérifier le comportement de votre application et valider qu'elle répond correctement aux cas inhabituels, qui sont difficiles à tester manuellement.

Vous pouvez utiliser la bibliothèque pour créer des tests unitaires locaux, qui vérifient généralement le comportement des classes de votre application qui interagissent avec le client Health Connect.

Le point d'entrée de la bibliothèque est la classe FakeHealthConnectClient, que vous utilisez dans les tests pour remplacer HealthConnectClient. Il présente les caractéristiques suivantes :

  • Représentation en mémoire des enregistrements, qui vous permet de les insérer, de les supprimer et de les lire
  • Génération de jetons de modification et suivi des modifications
  • Pagination pour les enregistrements et les modifications
  • Les réponses d'agrégation sont acceptées avec les stubs.
  • Autorise toute fonction à générer des exceptions
  • Un FakePermissionController pouvant être utilisé pour émuler les vérifications des autorisations

Modifications apportées à l'API

  • Ajout de FakeHealthConnectClient e8469
  • Ajouter des remplacements de stub pour FakeHealthConnectClient e8469

Nouvelles fonctionnalités

  • Ajout d'API expérimentales de dossier de santé personnel (DSP) pour lire et écrire des données médicales, basées sur le format FHIR® (Fast Healthcare Interoperability Resources). Les API DMP incluent les éléments suivants :
    • Constante FEATURE_PERSONAL_HEALTH_RECORD permettant de vérifier si les Renseignements sur la santé personnelle sont disponibles via l'API de disponibilité des fonctionnalités.
    • API pour les applications qui écrivent des sources de données et des dossiers médicaux.
    • API permettant aux applications de lire des sources et des dossiers de données médicales en fonction des autorisations accordées par l'utilisateur.
  • Ajout d'API expérimentales pour les enregistrements de séances de pleine conscience (I51c13), y compris :
    • API pour les applications qui écrivent des données de session de pleine conscience.
    • API pour les applications qui lisent les données de sessions de pleine conscience en fonction des autorisations accordées par l'utilisateur.

(I0f40d)
