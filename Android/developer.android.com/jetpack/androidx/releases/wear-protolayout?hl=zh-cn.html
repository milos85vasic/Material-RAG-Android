声明依赖项

如需添加 wear-protolayout 的依赖项，您必须将 Google Maven 制品库添加到项目中。如需了解详情，请参阅 Google 的 Maven 制品库。

在应用或模块的 build.gradle 文件中添加所需工件的依赖项：

Groovy

dependencies {
    // Use to implement support for Wear ProtoLayout Expressions
    implementation "androidx.wear.protolayout:protolayout-expression:1.3.0"

    // Use to implement support for Wear ProtoLayout
    implementation "androidx.wear.protolayout:protolayout:1.3.0"

    // Use to utilize components and layouts with Material design in your ProtoLayout
    implementation "androidx.wear.protolayout:protolayout-material:1.3.0"
}

Kotlin

dependencies {
    // Use to implement support for Wear ProtoLayout Expressions
    implementation("androidx.wear.protolayout:protolayout-expression:1.3.0")

    // Use to implement support for Wear ProtoLayout
    implementation("androidx.wear.protolayout:protolayout:1.3.0")

    // Use to utilize components and layouts with Material design in your ProtoLayout
    implementation("androidx.wear.protolayout:protolayout-material:1.3.0")
}

版本 1.3.0

2025 年 6 月 4 日

发布了 androidx.wear.protolayout:protolayout-*:1.3.0。版本 1.3.0 包含这些提交内容。

自 1.2.0 以来的重要变更

  • 手表采用 Material 3 设计，其中的组件和布局经过针对圆形显示屏的优化，可从小屏幕适当地扩缩到大屏幕
    • 其中包括仅限 Kotlin 的 protolayout-material3 库，该库针对以下组件和功能提供了更多类似 Compose 的 API：
    • 系统和表盘提供的动态配色主题，采用最新的 Material3 主题，用于设置颜色、形状和排版
    • MaterialScope，用于处理所有有主见的默认设置并简化自定义
    • iconEdgeButton、textEdgeButton
    • iconButton、textButton、button、imageButton、avatarButton、compactButton
    • titleCard、appCard、graphicDataCard、iconDataCard、textDataCard
    • circularProgressIndicator、segmentedCircularProgressIndicator
    • primaryLayout、buttonGroup
    • 所有组件都适用于任何 SDK 级别和 ProtoLayout 渲染程序版本，并在适用情况下提供合理的回退
  • 为基本 ProtoLayout 元素提供了更易于使用 Kotlin 的 Compose 风格 API
    • LayoutModifier，能够将大多数修饰符添加为链式函数（padding、contentDescription [包括 clearSemantics]、background、clip、opacity 等），可转换为现有的 Modifiers 对象
    • LayoutColors 和 LayoutString 作为类型，更易于使用动态字段和约束条件
    • text 和 fontStyle
    • 更好地支持 StateBuilder 映射，包括 DynamicDataMap 和 intAppDataKey 等工厂方法，以便更轻松地创建 AppDataKey 对象
  • Lottie 动画支持设置不同的触发器，以便在动画应开始时触发动画，例如在布局加载时或布局变为可见时
    • 改进了对渐变的支持：
    • Brush 中的线性渐变 API，可用于 Background 修饰符中的元素（例如 Box、Spacer 等）…
    • 允许在 ColorStop 中使用动态颜色值来实现线性渐变和扫描渐变
    • 弧形对象中的现有清除渐变现在支持动态颜色以及起始角和结束角
  • protolayout-expression 中的平台数据绑定，用于在布局的公开范围发生变化时随时接收信息，例如，可用于在用户滑动功能块时隐藏布局的某些部分
  • 添加了测试库 - protolayout-testing，以支持更轻松地对任何 ProtoLayout 元素进行单元测试覆盖
  • 新增了元素 DashedArcLine，并改进了一系列功能，使线条可以包含短划线，而现有的 ArcLine 则不支持
  • ArcSpacer 支持以 DP 尺寸（而非度数）设置其长度
  • 添加了适用于某些字体的圆润度 FontSetting 轴

版本 1.3.0-beta02

2025 年 5 月 7 日

发布了 androidx.wear.protolayout:protolayout-*:1.3.0-beta02。版本 1.3.0-beta02 包含这些提交内容。

bug 修复

  • 我们对适用于 API 36 及更高版本的排版设计值进行了重大改进。这是因为从 API 36 开始，所有功能块都将采用系统字体，因此这项更改可使功能块轮播界面更加一致。(If316f)
  • 如果未设置动态值的 layoutConstraints，Text、Spacer、ArcLine 和 DashedArcLine 构建器不会抛出异常。请注意，旧版渲染程序仍需要设置 layoutConstraints，并且会忽略未设置 layoutConstraints 的任何动态值。(Ic52e8)
  • 添加了标题语义修饰符，用于指明布局元素是出于无障碍目的而用于标题的内容部分，并默认将 primaryLayout 的标题槽中的文本标记为无障碍标题。(Iae1fb)
  • 对 primaryLayout 进行了最终的用户体验优化，在较小的屏幕上，标题槽和主槽之间的间距已缩小到 4dp（而非 6dp）。(I0e056)
  • 将默认内容描述应用于 textEdgeButton。(Ifaf8b)
  • 对 avatarButton 内部内边距进行了细微更新。(I0910b)
