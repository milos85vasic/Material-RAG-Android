Mendeklarasikan dependensi

Untuk menambahkan dependensi pada Benchmark, Anda harus menambahkan repositori Maven Google ke project Anda. Baca repositori Maven Google untuk informasi selengkapnya.

Macrobenchmark

Untuk menggunakan Macrobenchmark dalam project Anda, tambahkan dependensi berikut ke file build.gradle untuk modul macrobenchmark Anda:

Groovy

dependencies {
  androidTestImplementation "androidx.benchmark:benchmark-macro-junit4:1.4.1"
}

Kotlin

dependencies {
  androidTestImplementation("androidx.benchmark:benchmark-macro-junit4:1.4.1")
}

Microbenchmark

Untuk menggunakan Microbenchmark dalam project Anda, tambahkan dependensi berikut ke file build.gradle untuk modul microbenchmark Anda:

Groovy

dependencies {
    androidTestImplementation "androidx.benchmark:benchmark-junit4:1.4.1"
}

android {
    ...
    defaultConfig {
        ...
        testInstrumentationRunner "androidx.benchmark.junit4.AndroidBenchmarkRunner"
    }
}

Kotlin

dependencies {
    androidTestImplementation("androidx.benchmark:benchmark-junit4:1.4.1")
}

android {
    ...
    defaultConfig {
        ...
        testInstrumentationRunner = "androidx.benchmark.junit4.AndroidBenchmarkRunner"
    }
}

Library Microbenchmark juga menyediakan plugin Gradle untuk digunakan dengan modul microbenchmark. Plugin ini menyetel default konfigurasi build untuk modul, menyiapkan salinan output benchmark ke host, dan memberikan tugas ./gradlew lockClocks.

Untuk menggunakan plugin ini, sertakan baris berikut dalam blok `plugins` di file build.gradle level teratas:

Groovy

plugins {
  id 'androidx.benchmark' version '1.4.1' apply false
}

Kotlin

plugins {
  id("androidx.benchmark") version "1.4.1" apply false
}

Lalu, terapkan plugin ke file build.gradle modul benchmark Anda

Groovy

plugins {
  id 'androidx.benchmark'
}

Kotlin

plugins {
    id("androidx.benchmark")
}

Masukan

Masukan Anda membantu meningkatkan kualitas Jetpack. Beri tahu kami jika Anda menemukan masalah baru atau mempunyai masukan untuk meningkatkan kualitas library ini. Harap periksa masalah yang sudah diketahui dalam library ini sebelum membuat laporan baru. Anda dapat memberikan suara untuk masalah yang sudah diketahui dengan mengklik tombol bintang.

Lihat dokumentasi Issue Tracker untuk informasi selengkapnya.

Versi 1.4.1

androidx.benchmark:benchmark-*:1.4.1 dirilis. Versi 1.4.1 berisi commit ini.

Perbaikan Bug

  • Memperbaiki perfettoSdkTracing=true (misalnya, saat merekam aktivitas Compose dalam microbenchmark) akan menghentikan proses target jika sudah berjalan, dan StartupMode tidak ditentukan. (Ib2c1f)

Versi 1.4.0

30 Juli 2025

androidx.benchmark:benchmark-*:1.4.0 dirilis. Versi 1.4.0 berisi commit ini.

Perubahan penting sejak versi 1.3.0

Microbenchmark

  • Memindahkan tugas Gradle lockClocks dan unlockClocks ke project benchmark, bukan di level teratas untuk mematuhi isolasi project Gradle.
  • BenchmarkRule telah di-refactor untuk dibangun di atas coroutine, dan mendukung perilaku yield() yang lebih baik. Tindakan ini akan mengurangi risiko ANR secara signifikan selama menjalankan tolok ukur, terutama menjalankan CI yang panjang. Catatan: Benchmark UI harus dijalankan dengan measureRepeatedOnMainThread

Macrobenchmark

  • Solusi sementara yang ditambahkan di API 34+ untuk CompilationMode.None() akan memiliki performa yang tidak konsisten karena verifikasi ART sekarang mengompilasi sebagian aplikasi setelah peluncuran pertama.
  • Fitur eksperimental - Insight Startup dapat menandai masalah umum tertentu dalam Macrobenchmark startup dengan meneruskan MacrobenchmarkRule(..., experimentalConfig = ExperimentalConfig(StartupInsightsConfig(isEnabled = true))).
  • Menambahkan ArtMetric yang dapat digunakan untuk mendeteksi Kompilasi JIT, dan pemuatan class yang tidak dioptimalkan - keduanya berguna untuk memvalidasi pengoptimalan Profil Dasar Pengukuran.

Profil Dasar Pengukuran

  • BaselineProfileRule kini mengumpulkan profil untuk aplikasi multiproses.

Perubahan Lainnya

  • TraceProcessor telah dikeluarkan ke library-nya sendiri (androidx.benchmark:benchmark-traceprocessor) sehingga dapat digunakan di luar metrik Macrobenchmark, dalam kasus lain. Server ini juga dapat dijalankan di JVM Desktop dengan menentukan ServerLifecycleManager Anda sendiri.

Versi 1.4.0-alpha09

12 Maret 2025

androidx.benchmark:benchmark-*:1.4.0-alpha09 dirilis. Versi 1.4.0-alpha09 berisi commit ini.

Perubahan API

Hal ini juga merupakan langkah untuk memungkinkan penggunaan API Coroutine dan Java yang lebih mudah. Fungsi ekstensi ke TraceProcessor.runServer {} kini ditandai sebagai eksperimental, karena kemungkinan akan dipindahkan dan dijadikan non-eksperimental pada masa mendatang. (I358b4)

Perbaikan Bug

  • Memperbaiki pengambilan Profil Dasar Pengukuran dan Tolok Ukur yang tidak berfungsi dengan API 36 karena perubahan pada kotak mainan pgrep yang kini memerlukan -a untuk mencetak command line lengkap. (Idc991)
  • Memfilter konfigurasi perekaman aktivitas default untuk mengurangi risiko kehilangan data dalam rekaman aktivitas di level API yang lebih baru. (I54e8a)
  • Menambahkan argumen instrumentasi androidx.benchmark.killExistingPerfettoRecordings eksperimental, yang dapat disetel ke false agar memungkinkan kelanjutan pengambilan rekaman aktivitas Perfetto yang sudah ada. Secara default, perekaman aktivitas Perfetto yang sudah ada di perangkat akan dihentikan untuk mencegah gangguan. (I02a3c)
  • Kolom JSON context.osCodenameAbbreviated kini akan menjadi REL untuk versi OS yang dirilis pada dan di atas API 35, karena nama kode non-numerik tidak lagi didukung oleh platform yang mendasarinya. (Ib17fd)
  • Memperbaiki error di FrameTimingMetric saat frame disinkronkan ulang. (I7c6f4, b/394610806)
  • Tidak lagi mengasumsikan bahwa Choreographer#doFrame adalah frame stack teratas di thread utama untuk FrameTimingQuery. (Iee0e0, b/340206285)

Versi 1.4.0-alpha08

12 Februari 2025

    • Pelacakan metode pada beberapa versi Android OS dan ART akan memengaruhi fase pengukuran selanjutnya - pada versi ini, pelacakan metode dinonaktifkan secara default dan peringatan dicetak ke output Studio

  • Minifikasi
    • Menyematkan aturan proguard untuk meningkatkan kualitas microbenchmarking dengan minifikasi yang diaktifkan
    • Pengecilan/R8 dalam modul library memerlukan AGP 8.3, dan dapat diaktifkan melalui android.buildTypes.release.androidTest.enableMinification di build.gradle Anda
    • Menambahkan BlackHole.consume() API eksperimental untuk mencegah penghapusan kode tidak terpakai (If6812, b/286091643)

  • Plugin Gradle Profil Dasar Pengukuran baru mengotomatiskan pengambilan dan penyertaan profil dasar pengukuran dalam alur kerja pengujian dan build Anda.
  • BaselineProfileRule.collect kini stabil, versi API BaselineProfileRule.collectBaselineProfile eksperimental sebelumnya yang disederhanakan
    • Cukup tentukan packageName, dan jalankan aplikasi Anda
  • Untuk library yang membuat profil dasar pengukuran, Anda kini dapat memfilter aturan yang dibuat baik dalam kode (argumen BaselineProfileRule.collect), atau bahkan lebih sederhana di plugin Gradle
  • Perbaikan
    • Memperbaiki pengumpulan profil dasar pengukuran di Android U+ (Id1392, b/277645214)

Macrobenchmark

  • Kompilasi
    • Macrobenchmark kini mereset status kompilasi sepenuhnya dengan benar untuk setiap kompilasi - hal ini memerlukan penginstalan ulang APK sebelum Android 14, jadi tolok ukur di Android 14+ sangat direkomendasikan jika Anda ingin mempertahankan status (seperti login pengguna) dalam apa yang diukur.
    • Anda juga dapat mengatasi masalah ini dengan mengontrol kompilasi aplikasi secara terpisah, dan melewati kompilasi dengan argumen instrumentasi CompilationMode.Ignore() atau

  • Argumen Instrumentasi

    • Dukungan untuk argumen instrumentasi androidx.benchmark.dryRunMode.enable, (sudah tersedia di microbenchmark) untuk menjalankan validasi yang lebih cepat (misalnya, saat membuat benchmark, atau dalam pra-pengiriman)
    • Dukungan untuk androidx.benchmark.profiling.mode=StackSampling dan MethodTracing.
    • Menambahkan androidx.benchmark.enabledRules untuk memungkinkan pemfilteran runtime pengujian aturan profil dasar pengukuran vs. macrobenchmark
    • Menambahkan argumen androidx.benchmark.perfettoSdkTracing.enable untuk mengaktifkan pelacakan dengan tracing-perfetto, misalnya pelacakan rekomposisi Compose. Perhatikan bahwa jika digunakan dengan StartupMode.COLD, pengaturan waktu akan sangat terpengaruh karena library pelacakan dimuat dan diaktifkan selama startup aplikasi.

  • Persyaratan

    • Macrobenchmark kini memerlukan ProfileInstaller 1.3.0 atau yang lebih baru di aplikasi target, untuk mengaktifkan pengambilan / reset profil, dan penghapusan cache shader.

  • Metric API Eksperimental Baru

    • Menambahkan TraceSectionMetric eksperimental, yang memungkinkan ekstraksi waktu sederhana dari blok trace("") {} di aplikasi Anda, atau TraceMetric untuk memanfaatkan kemampuan kueri penuh TraceProcessor Perfetto.
    • Menambahkan PowerMetric eksperimental untuk merekam informasi penggunaan daya
    • Menambahkan MemoryCountersMetric eksperimental untuk menghitung kesalahan halaman
    • Menambahkan API PerfettoTraceProcessor eksperimental, yang digunakan secara internal untuk mengekstrak metrik dari rekaman aktivitas Sistem (alias rekaman aktivitas Perfetto)

  • Perbaikan

    • Memperbaiki error saat menginstal atau mengekstrak profil dari aplikasi yang diinstal dari beberapa APK (misalnya dari app bundle).
    • Memperbaiki FrameTimingMetric yang mengabaikan frame dengan ID frame yang tidak konsisten (umumnya, frame selama riak pada API 31+) (I747d2, b/279088460)
    • Memperbaiki error penguraian pada rekaman aktivitas > 64 MB (Ief831, b/269949822)
    • Memperjelas error saat image OS perangkat (terutama emulator) tidak dikonfigurasi dengan benar untuk pelacakan, atau kompilasi
    • Melewati pemeriksaan level baterai untuk perangkat tanpa baterai (mikro dan makro)
    • Peningkatan output file, dengan error yang lebih jelas untuk direktori output yang tidak valid, dan default yang lebih aman
    • Meningkatkan stabilitas StartupMode.COLD dengan menghapus cache shader secara konsisten (juga diekspos melalui MacrobenchmarkScope.dropShaderCache)
    • Memperbaiki fallback leanback untuk startActivityAndWait.

Microbenchmark

  • Fitur
    • Pembuatan profil dipindahkan ke fase terpisah, setelah metrik lainnya, sehingga satu eksekusi pengujian dapat menampilkan hasil pengaturan waktu dan pembuatan profil yang akurat.
  • API Eksperimental
    • Menambahkan API MicrobenchmarkConfig eksperimental untuk menentukan metrik kustom serta mengonfigurasi pelacakan dan pembuatan profil. Dapat digunakan untuk merekam rekaman aktivitas metode, atau merekam titik rekaman aktivitas (tetapi perhatikan overhead rekaman aktivitas).
    • Menambahkan API eksperimental untuk mengontrol BenchmarkState secara terpisah dari BenchmarkRule, tanpa JUnit
    • Menambahkan rekaman PerfettoTrace eksperimental untuk memungkinkan pengambilan rekaman aktivitas Perfetto, dengan konfigurasi kustom, terpisah dari API benchmark.
  • Perbaikan
    • Solusi untuk mengatasi tidak adanya spasi di awal output benchmark Android Studio.
    • Memperbaiki masalah saat peringatan gagal dicetak dalam output benchmark Android Studio.
    • Memperbaiki error SampledProfiling di Android 13 (API 33) dan yang lebih tinggi.
    • Peningkatan performa dryRunMode secara signifikan dengan melewati IsolationActivity dan perekaman aktivitas Perfetto (Hingga 10x lebih cepat dalam mode uji coba di versi OS yang lebih lama).

    • Error ini dapat dikembalikan menjadi peringatan dengan argumen instrumentasi. Contoh: -e androidx.benchmark.suppressErrors "DEBUGGABLE,LOW_BATTERY"

  • Error saat menulis ke penyimpanan eksternal pada perangkat Android Q memberikan pesan yang lebih deskriptif, disertai saran cara menyelesaikan masalah tersebut
