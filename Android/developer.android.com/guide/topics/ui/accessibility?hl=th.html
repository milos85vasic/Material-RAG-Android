ข้อมูลจากธนาคารโลกระบุว่าประชากรโลก 15% มีลักษณะความพิการบางประเภท ผู้พิการจะใช้แอปและบริการที่รองรับการช่วยเหลือพิเศษเพื่อสื่อสาร เรียนรู้ และทำงาน การทำให้แอปเข้าถึงได้จะช่วยให้คุณเข้าถึงผู้ใช้เหล่านี้ได้

ปรับปรุงความอเนกประสงค์ของแอป

ฟีเจอร์การช่วยเหลือพิเศษมีประโยชน์ต่อผู้ใช้ทุกคน เช่น หากมีคนใช้แอปของคุณขณะทำอาหาร บุคคลนั้นจะใช้คำสั่งเสียงแทนท่าทางสัมผัสเพื่อไปยังส่วนต่างๆ ได้ ฟีเจอร์สำหรับผู้มีสายตาเลือนรางช่วยให้แอปมีประโยชน์แม้ในวันที่แดดจัด

ปฏิบัติตามข้อกำหนดของรัฐบาลและสถาบัน

ปัจจุบันหลายประเทศกำหนดให้ผลิตภัณฑ์ทั้งหมดที่พนักงานภาครัฐใช้ต้องเข้าถึงได้ และบริษัทต่างๆ ก็เริ่มดำเนินการตาม

การเริ่มต้นใช้งาน

ดูข้อมูลเกี่ยวกับคอนทราสต์ของสี ขนาดเป้าหมายการสัมผัส ป้ายกำกับเนื้อหา และแนวทางปฏิบัติอื่นๆ ที่สร้างความแตกต่างอย่างมากให้แก่ผู้ใช้ ดูความหมายของการสร้างแอปที่เข้าถึงได้บน Android
เส้นทางการช่วยเหลือพิเศษ

การออกแบบ

การสร้างประสบการณ์ที่เข้าถึงได้ต้องเริ่มต้นตั้งแต่ต้น การช่วยเหลือพิเศษในการออกแบบช่วยให้ผู้ใช้ที่มีความสามารถหลากหลายไปยังส่วนต่างๆ ทำความเข้าใจ และใช้ UI ได้
ออกแบบแอปที่เข้าถึงได้

พัฒนา

ขณะสร้างแอป ให้ทำตามหลักการพื้นฐานด้านการช่วยเหลือพิเศษที่ส่งผลอย่างมากต่อผู้ใช้ เช่น การอธิบายองค์ประกอบ UI แต่ละรายการอย่างกระชับ และตรวจสอบว่าผู้ใช้เข้าถึงขั้นตอนทั้งหมดได้
พัฒนาแอปที่เข้าถึงได้

ทดสอบ

ใช้หลักเกณฑ์ เช่น Material Design, เครื่องมือ เช่น เครื่องตรวจการช่วยเหลือพิเศษ และการตรวจสอบอัตโนมัติเพื่อหาวิธีปรับปรุงการช่วยเหลือพิเศษของแอป
การทดสอบการช่วยเหลือพิเศษ
