Declarar dependências

Para adicionar uma dependência à Core, adicione o repositório Maven do Google ao seu projeto. Leia mais informações no Repositório Maven do Google.

Adicione as dependências dos artefatos necessários no arquivo build.gradle para seu app ou módulo:

Groovy

dependencies {
    def core_version = "1.17.0"

    // Java language implementation
    implementation "androidx.core:core:$core_version"
    // Kotlin
    implementation "androidx.core:core-ktx:$core_version"

    // To use RoleManagerCompat
    implementation "androidx.core:core-role:1.1.0"

    // To use the Animator APIs
    implementation "androidx.core:core-animation:1.0.0"
    // To test the Animator APIs
    androidTestImplementation "androidx.core:core-animation-testing:1.0.0"

    // Optional - To enable APIs that query the performance characteristics of GMS devices.
    implementation "androidx.core:core-performance:1.0.0"

    // Optional - to use ShortcutManagerCompat to donate shortcuts to be used by Google
    implementation "androidx.core:core-google-shortcuts:1.1.0"

    // Optional - to support backwards compatibility of RemoteViews
    implementation "androidx.core:core-remoteviews:1.1.0"

    // Optional - APIs for SplashScreen, including compatibility helpers on devices prior Android 12
    implementation "androidx.core:core-splashscreen:1.2.0-rc01"
}

Core-Viewtree: versão 1.0

Versão 1.0.0

26 de fevereiro de 2025

androidx.core:core-viewtree:1.0.0 foi promovido à primeira versão estável sem mudanças desde a versão RC anterior.

Essa biblioteca apresenta o conceito de uma visualização que pode ter um elemento pai independente. Um elemento pai independente de uma visualização é um objeto View separado que atua como o elemento pai da visualização, mas não é definido pela propriedade View.parent. Exemplos de visualizações com elementos pai desconexos são ViewOverlays, pop-ups e caixas de diálogo, que aparecem fora da hierarquia principal de visualizações. Uma visualização só pode ter um pai independente se não tiver um pai direto pela propriedade View.parent da plataforma. No momento, o androidx só define o elemento pai independente para ViewOverlays criado por Transition 1.6.0-alpha01 e versões mais recentes. Você também pode especificar seus próprios elementos principais disjuntos para visualizações. Consulte a documentação para mais informações.

  • Funcionalidade de formatação de data e hora seguindo as melhores práticas atuais, respeitando as configurações personalizadas do usuário e fornecendo uma API unificadora que funciona corretamente em versões antigas e novas do Android.

O componente será o veículo para fazer backport de novas APIs i18n, adicionar novas funcionalidades ou correções de i18n e, em geral, facilitar a internacionalização.
