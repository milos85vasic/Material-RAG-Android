Compose Material 3 Adaptive

  
API-Referenz
androidx.compose.material3.adaptive
Mit der adaptiven Material 3-Bibliothek können Sie adaptive UIs erstellen, die sich automatisch an die aktuellen Fensterkonfigurationen wie Fenstergrößenklassen oder Gerätestatus anpassen. Die Bibliothek bietet sowohl Standard-Scaffold-Implementierungen als auch die erforderlichen Composables für Bausteine, mit denen Sie Ihre eigenen benutzerdefinierten Funktionen erstellen können.
Letzte Aktualisierung Stabile Version Releasekandidat Beta-Release Alpha-Release
27. August 2025 1.1.0 - 1.2.0-beta01 -

Abhängigkeiten deklarieren

Wenn Sie eine Abhängigkeit von Compose hinzufügen möchten, müssen Sie Ihrem Projekt das Google Maven-Repository hinzufügen. Weitere Informationen finden Sie im Maven-Repository von Google.

Fügen Sie der Datei build.gradle für Ihre App oder Ihr Modul die Abhängigkeiten für die benötigten Artefakte hinzu:

Groovy

dependencies {
    implementation "androidx.compose.material3.adaptive:adaptive:1.2.0-beta01"
    implementation "androidx.compose.material3.adaptive:adaptive-layout:1.2.0-beta01"
    implementation "androidx.compose.material3.adaptive:adaptive-navigation:1.2.0-beta01"
}

Kotlin

dependencies {
    implementation("androidx.compose.material3.adaptive:adaptive:1.2.0-beta01")
    implementation("androidx.compose.material3.adaptive:adaptive-layout:1.2.0-beta01")
    implementation("androidx.compose.material3.adaptive:adaptive-navigation:1.2.0-beta01")
}

Weitere Informationen zu Abhängigkeiten finden Sie unter Build-Abhängigkeiten hinzufügen.

Feedback

Ihr Feedback hilft uns, Jetpack zu verbessern. Lassen Sie es uns wissen, wenn Sie neue Probleme entdecken oder Ideen zur Verbesserung dieser Bibliothek haben. Sehen Sie sich die vorhandenen Probleme in dieser Bibliothek an, bevor Sie ein neues erstellen. Sie können für ein vorhandenes Problem abstimmen, indem Sie auf die Schaltfläche mit dem Stern klicken.

Compose Material3 Adaptive Navigation3 Version 1.0

Version 1.0.0-alpha02

10. September 2025

androidx.compose.material3.adaptive:adaptive-navigation3:1.0.0-alpha02 und androidx.compose.material3.adaptive:adaptive-navigation3-android:1.0.0-alpha02 werden veröffentlicht. Version 1.0.0-alpha02 enthält diese Commits.

Version 1.0.0-alpha01

27. August 2025

androidx.compose.material3.adaptive:adaptive-navigation3:1.0.0-alpha01 und androidx.compose.material3.adaptive:adaptive-navigation3-android:1.0.0-alpha01 werden veröffentlicht. Version 1.0.0-alpha01 enthält diese Commits.

Neue Funktionen

Wir freuen uns, die erste Alphaversion der adaptive-navigation3-Bibliothek anzukündigen. Diese neue Bibliothek ist Teil des Material Adaptive- und Navigation3-Ökosystems und soll Ihnen dabei helfen, adaptive Material-UIs zu erstellen, die in die Navigation3-Bibliothek integriert sind.

Verwenden Sie Szenenstrategien, um ein kanonisches Gerüst für die Liste/Detailansicht (ListDetailSceneStrategy, rememberListDetailSceneStrategy) oder ein unterstützendes Gerüst (SupportingPaneSceneStrategy, rememberSupportingPaneSceneStrategy) zu implementieren. Weitere Informationen zu Szenen, Szenenstrategien und anderen neuen Konzepten, die in Navigation 3 eingeführt wurden, finden Sie im Leitfaden zu Navigation 3.

Compose Material3 Adaptive Version 1.2

Version 1.2.0-beta02

10. September 2025

androidx.compose.material3.adaptive:adaptive-*:1.2.0-beta02 wird veröffentlicht. Version 1.2.0-beta02 enthält diese Commits.

API-Änderungen

  • Die Funktion zum Ziehen zum Anpassen der Größe wird aus der öffentlichen API entfernt (Ic85ba, b/437953743, b/442636084).
  • Benennen Sie Scrim() in LevitatedPaneScrim() um und blenden Sie Attribute der Klassen „Levitated“ und „Reflowed“ aus. (I090e1, b/427953101)
  • calculatePosture() API als intern ausblenden (Ie7227, b/424442112)

Fehlerkorrekturen

  • Neuen anfänglichen Anker verwenden, wenn sich die Ankerliste für die Bereichserweiterung ändert (I91cd1, b/438829477)

Version 1.2.0-beta01

27. August 2025

androidx.compose.material3.adaptive:adaptive-*:1.2.0-beta01 wird veröffentlicht. Version 1.2.0-beta01 enthält diese Commits.

Neue Funktionen

  • Unterstützung alternativer Eingabemethoden für die Funktion zum Ziehen zum Anpassen der Größe (Id7796)

API-Änderungen

  • Unterstützung von benutzerdefiniertem Ziehverhalten mit dem Status der Bereichsmaximierung (If5c61)
  • Fügt in der Benutzeroberfläche des Bereichs einen Indikator hinzu, um anzugeben, ob ein Bereich interaktiv sein soll oder nicht. Außerdem wird dieser Indikator verwendet, um den Zugriff auf Bedienungshilfen für zugrunde liegende Bereiche zu deaktivieren, wenn ein schwebender Bereich mit einem Scrim angezeigt wird. (If36f3)
  • Scrim-Definition durch zusammensetzbare Lambdas ersetzen (I7d811)
  • Markieren Sie APIs für Fenstergröße und Körperhaltung als experimentell (I4ee96).
  • PaneScaffoldHorizontalOrder als versiegelte öffentliche API (Ia4ebe) verfügbar machen

Fehlerkorrekturen

  • Absturz aufgrund unnötiger Annäherungsmessung umgehen (I0a65a, b/418932957)

Version 1.2.0-alpha11

13. August 2025

androidx.compose.material3.adaptive:adaptive-*:1.2.0-alpha11 wird veröffentlicht. Version 1.2.0-alpha11 enthält diese Commits.

API-Änderungen

  • Einführung einer generischen PaneScaffoldRole-Schnittstelle zur Verbesserung der API-Lesbarkeit (I1b757)

  • Das Problem wurde behoben, dass anfängliche Anker nicht berücksichtigt wurden (I32f5d, b/418296559).
