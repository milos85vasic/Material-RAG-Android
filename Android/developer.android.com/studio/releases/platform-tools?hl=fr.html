Notes de version du composant SDK Platform Tools
Restez organisé à l'aide des collections Enregistrez et classez les contenus selon vos préférences.

SDK Tools pour Android est un composant du SDK Android. Il comprend des outils qui offrent une interface avec la plate-forme Android, principalement adb et fastboot. Bien que adb soit nécessaire au développement d'applications Android, les développeurs d'applications se contentent généralement d'utiliser la copie installée avec Studio. Ce téléchargement peut s'avérer utile si vous souhaitez utiliser adb directement à partir de la ligne de commande et que Studio n'est pas installé. (Si Studio est installé, il vous suffit d'utiliser la copie associée, car Studio la met automatiquement à jour.) fastboot est nécessaire si vous souhaitez déverrouiller le programme bootloader de l'appareil et le flasher avec une nouvelle image système. Ce package contenait auparavant systrace, mais celui-ci a été remplacé par Studio Profiler, gpuinspector.dev ou Perfetto.

Parmi les nouvelles fonctionnalités de adb et fastboot, certaines ne sont disponibles qu'avec les versions récentes d'Android, mais comme elles sont rétrocompatibles, seule la dernière version du composant SDK Platform-Tools est nécessaire. Contentez-vous de signaler les bugs si vous rencontrez des exceptions.

Téléchargements

Si vous êtes un développeur Android, vous devez vous procurer le composant SDK Platform-Tools le plus récent à partir du SDK Manager d'Android Studio ou de l'outil de ligne de commande sdkmanager. Le composant est ainsi enregistré à l'emplacement adéquat, avec les autres composants du SDK Tools pour Android, et il est facilement mis à jour.

Révisions

36.0.0 (avr. 2025)

  • adb
    • Réécriture du backend USB libusb (utilise l'API synchrone au lieu de l'API asynchrone). Améliore la fiabilité et corrige l'épuisement de la mémoire sous Linux.
    • Le branchement à chaud du backend USB Libusb est désormais compatible avec Windows (permet la détection de la vitesse USB).
    • L'icône server-status améliorée s'affiche désormais si le service mDNS est activé.
    • Correction d'un bug macOS qui empêchait l'envoi de paquets de longueur nulle, ce qui entraînait un blocage de la connexion (b/208675141).
    • Correction d'un bug "use-after-free" dans le backend libusb.

35.0.2 (juillet 2024)

  • adb
    • Correction du bug du backend mDNS openscreen qui entraînait l'arrêt du serveur lors d'une requête tronquée problème 294120933.
    • Faire fonctionner le backend mDNS openscreen sur macOS.
    • Définissez le backend mDNS openscreen par défaut sur toutes les plates-formes.
    • Prise en charge de la détection de l'USB SuperSpeed+ (vitesses actuelles et négociées) à des fins de diagnostic.
    • Arrêt progressif : libérez toutes les interfaces USB à l'arrêt (tous les OS).

35.0.1 (mars 2024)

  • adb
    • Passer à libusb 1.0.27

35.0.0 (février 2024)

  • adb
    • Passer à libusb comme valeur par défaut sous Linux problème 270205252.
    • Correction du démarrage d'adb sur les hôtes sans USB.
    • Correction des blocages adb causés par des appareils USB qui signalent incorrectement des descripteurs de longueur nulle problème 302212871.
    • Correction du code de retour de adb shell lorsque l'appareil se déconnecte problème 321787891.
  • fastboot
    • Limitez la taille maximale de la file d'attente des paquets entrants.
    • Suppression des goulots d'étranglement qui limitaient auparavant la vitesse de téléchargement à environ 120 Mo/s. Fastboot peut désormais saturer un bus SuperSpeed+ et atteindre des vitesses allant jusqu'à 980 Mo/s, selon l'appareil.

34.0.5 (octobre 2023)

  • adb
    • adb utilise désormais libusb par défaut sur macOS pour résoudre le problème 270205252.
    • Auparavant, adb répondait avec un code de réussite lorsque l'association sans fil échouait. Ce problème a été résolu en renvoyant un code d'échec (1) et une erreur visible par l'utilisateur (error: protocol fault (couldn't read status message...)). echo $? affiche désormais 1.
    • adb wait-for-disconnect est désormais opérationnel pour le débogage sans fil (non USB).
    • Ajout d'une nouvelle interface DbC pour la future compatibilité de ChromeOS avec adb.
  • fastboot
    • Correction de flashall sur les appareils Pixel 3.

34.0.4 (juillet 2023)

  • adb
    • Propager -a (gListenAll) lorsque adb duplique un serveur hôte adb (auparavant, l'indicateur ne fonctionnait que pour adb -a server nodaemon)
    • Root et déroot plus rapides
    • Reland Flag(env) guarding clear endpoint (device) feature for OSX usb start. (problème 270205252).
  • fastboot
    • Mac : supprimer les nouvelles tentatives sur l'itérateur d'E/S non valide (échec clignotant avec LIBUSB_TRANSFER_CANCELLED)
    • Windows : résoudre l'erreur "Sparse file is too large or invalid" (Fichier sparse trop volumineux ou non valide) lors de l'utilisation de "flashall"
    • Toutes les plates-formes : correction de l'erreur "ANDROID_PRODUCT_OUT not set" lors de l'utilisation de "update"
