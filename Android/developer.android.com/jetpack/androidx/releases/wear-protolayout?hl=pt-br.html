wear protolayout

  
Referência da API
androidx.wear.protolayout
Essa biblioteca permite definir um conjunto de layouts de interface e expressões que não são da interface para renderização/avaliação em plataformas remotas.

Declarar dependências

Para adicionar uma dependência à biblioteca wear-protolayout, adicione o repositório Maven do Google ao seu projeto. Consulte a seção Repositório Maven do Google para mais informações.

Adicione as dependências dos artefatos necessários ao arquivo build.gradle do seu app ou módulo:

Groovy

dependencies {
    // Use to implement support for Wear ProtoLayout Expressions
    implementation "androidx.wear.protolayout:protolayout-expression:1.3.0"

    // Use to implement support for Wear ProtoLayout
    implementation "androidx.wear.protolayout:protolayout:1.3.0"

    // Use to utilize components and layouts with Material design in your ProtoLayout
    implementation "androidx.wear.protolayout:protolayout-material:1.3.0"
}

Kotlin

dependencies {
    // Use to implement support for Wear ProtoLayout Expressions
    implementation("androidx.wear.protolayout:protolayout-expression:1.3.0")

    // Use to implement support for Wear ProtoLayout
    implementation("androidx.wear.protolayout:protolayout:1.3.0")

    // Use to utilize components and layouts with Material design in your ProtoLayout
    implementation("androidx.wear.protolayout:protolayout-material:1.3.0")
}

Para mais informações sobre dependências, consulte Adicionar dependências de build.

Versão 1.3.0

4 de junho de 2025

Lançamento de androidx.wear.protolayout:protolayout-*:1.3.0. A versão 1.3.0 contém estas confirmações.

Mudanças importantes desde a versão 1.2.0

  • O design do Material 3 chega ao relógio, com componentes e layouts otimizados para a tela redonda e dimensionados adequadamente de tamanhos pequenos a grandes
    • Isso inclui a biblioteca protolayout-material3 exclusiva do Kotlin com mais APIs semelhantes ao Compose para os seguintes componentes e recursos:
    • Tema de cores dinâmico do sistema e mostrador do relógio com o tema mais recente do Material 3 para cores, formas e tipografia
    • MaterialScope para cuidar de todos os padrões opinativos e facilitar a personalização
    • iconEdgeButton, textEdgeButton
    • iconButton, textButton, button, imageButton, avatarButton, compactButton
    • titleCard, appCard, graphicDataCard, iconDataCard, textDataCard
    • circularProgressIndicator, segmentedCircularProgressIndicator
    • primaryLayout, buttonGroup
    • Todos os componentes funcionam em todos os níveis do SDK e na versão do renderizador ProtoLayout, fornecendo substitutos adequados quando aplicável
  • APIs mais compatíveis com o Kotlin e semelhantes ao Compose para elementos básicos do ProtoLayout
    • LayoutModifier com capacidade de adicionar a maioria dos modificadores como funções conectadas (padding, contentDescription (incluindo clearSemantics), background, clip, opacity etc.), convertíveis para o objeto Modifiers
    • LayoutColors e LayoutString como tipos com suporte mais fácil para usar campos e restrições dinâmicos
    • text e fontStyle
    • Melhor suporte de mapeamento para StateBuilder, incluindo DynamicDataMap e métodos de fábrica, como intAppDataKey, para facilitar a criação de objetos AppDataKey.
  • As animações Lottie oferecem suporte à opção de definir diferentes acionadores quando a animação precisa ser iniciada, por exemplo, quando o layout é carregado ou quando o layout fica visível
    • Suporte aprimorado para gradientes:
    • API de gradiente linear como parte de Brush que pode ser usada em modificadores Background para elementos como Box, Spacer etc. …
    • Permitir valores de cor dinâmicos em ColorStop usados para gradiente linear e de varredura
    • O gradiente de varredura em objetos de arco agora oferece suporte a cores dinâmicas e ângulos iniciais e finais
  • Vinculação de dados da plataforma em protolayout-expression para receber informações sempre que a visibilidade do layout for alterada. Por exemplo, isso pode ser usado para ocultar determinadas partes do layout enquanto o bloco é deslizado para
  • A biblioteca de testes, protolayout-testing, foi adicionada para oferecer suporte a uma cobertura de teste de unidade mais fácil para qualquer elemento ProtoLayout.
  • Novo elemento DashedArcLine com um conjunto aprimorado de recursos para que a linha possa ter traços, ao contrário do ArcLine
  • Suporte a ArcSpacer para definir o comprimento na dimensão DP em vez de graus
  • Adição do eixo de arredondamento FontSetting aplicável a algumas fontes

Versão 1.3.0-beta02

7 de maio de 2025

Lançamento de androidx.wear.protolayout:protolayout-*:1.3.0-beta02. A versão 1.3.0-beta02 contém estas confirmações.

Correções de bugs

  • Fizemos uma melhoria importante nos valores de design de tipografia aplicados na API 36 e versões mais recentes. Isso ocorre porque, a partir da API 36, todos os Blocos serão na fonte do sistema. Essa mudança aumenta a consistência no carrossel de Blocos. (If316f).
  • Os builders Text,Spacer, ArcLine e DashedArcLine não vão gerar erros se o layoutConstraints para valores dinâmicos não estiver definido. Os renderizadores mais antigos ainda exigem que o layoutConstraints seja definido e ignoram qualquer valor dinâmico que não tenha sido definido. (Ic52e8).
  • Adicionamos o modificador de semântica de título para indicar que um elemento de layout é um título de uma seção de conteúdo para fins de acessibilidade e marcamos o texto no slot de título de primaryLayout como título de acessibilidade por padrão. (Iae1fb).
  • Otimização final da UX do primaryLayout, em que o espaço entre o slot de título e o slot principal foi reduzido para 4 dp em vez de 6 dp em telas menores. (I0e056).
  • Aplique a descrição de conteúdo padrão a textEdgeButton. (Ifaf8b).
  • Pequena atualização no padding interno de avatarButton. (I0910b).

Versão 1.3.0-alpha10

12 de março de 2025

Lançamento de androidx.wear.protolayout:protolayout-*:1.3.0-alpha10. A versão 1.3.0-alpha10 contém estas confirmações.

Novos recursos

  • Foi adicionado um auxiliar para o pincel no modificador de segundo plano do Kotlin. (I995de).
  • O campo obrigatório do contexto do Android foi divulgado em MaterialScope para facilitar o uso em funções de desenvolvedores que estão criando componentes para blocos do Material3. (I7df73).

Mudanças na API

  • Renomeação da API platformVisibilityStatus para PlatformEventSources.isLayoutVisible e adição de uma nova API experimental PlatformEventSources.isLayoutUpdatePending. (Ie1e04).

Correções de bugs

  • As margens para primaryLayout agora são arredondadas corretamente, o que pode afetar alguns layouts com até 2 dp a menos de espaço para o slot principal. (I8f5d3).
