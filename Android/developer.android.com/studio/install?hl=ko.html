Android 스튜디오 설치
컬렉션을 사용해 정리하기 내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

참고: Android 스튜디오의 Gemini 및 Firebase Crashlytics와 같은 서비스 통합은 최신 안정 채널 버전의 Android 스튜디오와 지난 10개월 동안 출시된 주요 버전 (패치 포함)에서만 사용할 수 있습니다. 이전 버전의 Android 스튜디오를 사용하는 경우 Cloud 서비스에 액세스하려면 업데이트해야 합니다. 자세한 내용은 Android 스튜디오 및 클라우드 서비스 호환성을 참고하세요.

클릭 몇 번으로 Android 스튜디오를 설정할 수 있습니다. 먼저 시스템 요구사항을 확인하세요. 그런 다음 최신 버전의 Android 스튜디오를 다운로드합니다.

Android Emulator를 사용하면 실제 기기가 없어도 다양한 기기 구성과 Android API 수준에서 앱을 테스트할 수 있습니다. 이러한 이유로 Android 앱을 테스트하는 데 권장되는 방법입니다. 자세한 내용은 Android Emulator에서 앱 실행을 참고하세요.

하지만 최신 실제 Android 기기 또는 Android 기기 스트리밍 서비스를 사용하여 에뮬레이터 없이 Android 스튜디오를 사용하여 앱을 배포하고 테스트할 수도 있습니다.

시스템에 리소스가 제한적인 경우 고급 로컬 하드웨어 없이도 강력한 개발 환경을 제공하는 클라우드 기반 IDE 솔루션인 IDX의 Android 스튜디오를 고려해 보세요.

Android 스튜디오 및 Android Emulator가 포함된 Android 스튜디오의 최소 요구사항

나열된 최소 요구사항을 사용하면 Android 스튜디오를 시작할 수 있으며 일반적으로 다음 사용 사례에 적용됩니다.

  • 소규모 프로젝트 및 샘플 생성 및 지원

  • 단일 Android Virtual Device (AVD)를 만듭니다.

  • 단일 AVD, 로컬 실제 기기 또는 Android 기기 스트리밍 기기에 앱을 배포합니다. XR AVD는 여기에 포함되지 않습니다.

참고: Android 스튜디오를 단독으로 실행할지 아니면 Android 스튜디오를 Android Emulator와 함께 실행할지에 따라 일부 최소 요구사항이 다릅니다. 이러한 경우 두 요구사항이 모두 나열되고 스튜디오 또는 스튜디오 및 에뮬레이터로 적절하게 라벨이 지정됩니다.

Android 스튜디오와 Android Emulator를 실행하는 것이 좋습니다.

Android 스튜디오와 Android 에뮬레이터를 실행하기 위한 권장 시스템 사양은 다음과 같습니다. 일반적으로 다음 사용 사례가 포함됩니다.

  • 더 크고 복잡한 코드베이스에 관한 전문성 개발
참고: 프로젝트가 클수록 IDE에서 컴파일하고 로드하는 데 더 높은 RAM 및 CPU 사양이 필요합니다.
  • XR 기기를 비롯한 여러 Android Virtual Device (AVD) 만들기
참고: 생성된 추가 AVD마다 최대 6GB의 추가 스토리지가 필요할 수 있습니다.
  • XR 기기를 포함한 여러 AVD를 동시에 실행합니다.
참고: 여러 AVD를 동시에 실행하려면 AVD당 약 4GB의 메모리가 필요합니다. 정확한 금액은 시스템에 따라 다를 수 있습니다.

64비트 시스템에 필요한 라이브러리

64비트 버전의 Ubuntu를 실행하는 경우, 다음 명령으로 일부 32비트 라이브러리를 설치해야 합니다.

sudo apt-get install libc6:i386 libncurses5:i386 libstdc++6:i386 lib32z1 libbz2-1.0:i386

64비트 Fedora를 실행하는 경우, 명령어는 다음과 같습니다.

sudo yum install zlib.i686 ncurses-libs.i686 bzip2-libs.i686

ChromeOS

Android 스튜디오와 Android Emulator를 지원하기 위한 시스템 요구사항은 ChromeOS 문서의 Android 개발을 참고하세요.

ChromeOS에 Android 스튜디오를 설치하려면 다음 단계를 따르세요.

  1. ChromeOS용 Linux를 설치합니다.
  2. 파일 앱을 열고 내 파일의 다운로드 폴더에서 DEB 패키지를 찾습니다.

  3. DEB 패키지를 마우스 오른쪽 버튼으로 클릭하고 Linux(베타)로 설치를 선택합니다.

    • 이전에 Android 스튜디오를 설치한 경우 이전 Android 스튜디오 설정을 가져올지 여부를 선택한 다음 OK를 클릭합니다.

  4. 개발에 필요한 Android SDK 구성요소를 다운로드하는 작업이 포함된 Android 스튜디오 설정 마법사를 완료합니다.

  5. 설치가 완료되면 런처 또는 ChromeOS Linux 터미널에서 Android 스튜디오를 실행합니다. 기본 설치 디렉터리 /opt/android-studio/bin에서 studio를 실행합니다.

새로운 도구와 기타 API를 사용할 수 있게 되면 Android 스튜디오에서 대화상자로 알려줍니다. 업데이트를 수동으로 확인하려면 Help > Check for Update를 클릭합니다.

참고: ChromeOS에서 Android 스튜디오를 사용하는 경우 현재 연결된 하드웨어 기기에만 앱을 배포할 수 있습니다. 자세한 내용은 하드웨어 기기에서 앱 실행을 참고하세요.

이 페이지에 나와 있는 콘텐츠와 코드 샘플에는 콘텐츠 라이선스에서 설명하는 라이선스가 적용됩니다. 자바 및 OpenJDK는 Oracle 및 Oracle 계열사의 상표 또는 등록 상표입니다.
