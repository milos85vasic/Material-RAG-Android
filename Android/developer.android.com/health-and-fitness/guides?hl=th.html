เริ่มต้นใช้งานสุขภาพบน Android
จัดทุกอย่างให้เป็นระเบียบอยู่เสมอด้วยคอลเล็กชัน บันทึกและจัดหมวดหมู่เนื้อหาตามค่ากำหนดของคุณ

Android Health มี API ต่อไปนี้เพื่อสร้างแอปสุขภาพและ ฟิตเนสในอุปกรณ์รูปแบบต่างๆ ได้แก่ Health Connect และ บริการด้านสุขภาพบน Wear OS คุณจะใช้เพียงอย่างเดียวหรือทั้ง 2 อย่างสำหรับ ประสบการณ์การใช้งานแอปก็ได้

ศูนย์นักพัฒนาแอปนี้มีคำแนะนำในการใช้ทั้ง 2 API และชี้ไปยัง แหล่งข้อมูลเพิ่มเติมจากทั่วทั้ง Android เพื่อช่วยคุณสร้างแอปสุขภาพและการออกกำลังกาย

อ่านและเขียนข้อมูลสุขภาพและการออกกำลังกายในอุปกรณ์โดยใช้ Health Connect

Health Connect เป็นแพลตฟอร์ม Android ที่อนุญาตให้แอปสุขภาพและการออกกำลังกายจัดเก็บและแชร์ข้อมูลในอุปกรณ์เดียวกันภายในระบบนิเวศแบบรวม และยังให้ผู้ใช้ควบคุมแอปที่สามารถอ่านและเขียนข้อมูลสุขภาพและการออกกำลังกายได้จากที่เดียวด้วย Health Connect รองรับการอ่านและการเขียนข้อมูลที่แตกต่างกันกว่า 50 ประเภท ตั้งแต่ความเร็วในการปั่นจักรยานไปจนถึงอุณหภูมิร่างกาย หากคุณกำลังสร้างแอปใหม่ เราขอแนะนำให้ใช้ Health Connect

เริ่มต้นใช้งาน Health Connect

เข้าถึงข้อมูลเซ็นเซอร์คุณภาพสูงด้วยบริการด้านสุขภาพบน Wear OS

บริการด้านสุขภาพบน Wear OS คือ API ที่ทำหน้าที่เป็นตัวกลางระหว่างเซ็นเซอร์ต่างๆ และอัลกอริทึมที่เกี่ยวข้อง ในอุปกรณ์ Wear OS API จะให้ข้อมูลคุณภาพสูงที่เกี่ยวข้องกับกิจกรรม การออกกำลังกาย และสุขภาพแก่แอปในลักษณะที่ประหยัดแบตเตอรี่ บริการข้อมูลสุขภาพจะสอดคล้องกันในอุปกรณ์ที่ใช้ Wear OS 3 ขึ้นไป ซึ่งหมายความว่าคุณ ต้องเขียนแอปเพียงครั้งเดียว และบริการข้อมูลสุขภาพจะดูแลให้แอป ทำงานเหมือนกันไม่ว่าจะเป็นอุปกรณ์ใดก็ตาม

เริ่มต้นใช้งานบริการด้านสุขภาพ

Android Health ในอุปกรณ์ต่างๆ

Health Connect พร้อมใช้งานในอุปกรณ์เคลื่อนที่ Android เท่านั้น อย่างไรก็ตาม หากคุณ มีแหล่งข้อมูลอื่นๆ ที่เชื่อมต่อกับแอปบนอุปกรณ์เคลื่อนที่ เช่น อุปกรณ์ที่สวมใส่ได้ คุณสามารถใช้แอปบนอุปกรณ์เคลื่อนที่เพื่ออำนวยความสะดวกในการโอนข้อมูลจากอุปกรณ์ที่สวมใส่ได้ ไปยัง Health Connect และจาก Health Connect ไปยังอุปกรณ์ที่สวมใส่ได้

หมายเหตุ: Health Connect ไม่เหมาะกับการให้ข้อมูลแบบเรียลไทม์ แต่เหมาะกับการ จัดเก็บและรวมข้อมูลในแอปและอุปกรณ์ต่างๆ

ขณะนี้ Health Services พร้อมให้บริการสำหรับอุปกรณ์ Wear OS ที่ใช้ Wear OS 3 ขึ้นไปเท่านั้น สำหรับอุปกรณ์อื่นๆ ทั้งหมด รวมถึงโทรศัพท์ คุณควรดูเอกสารประกอบสำหรับ SensorManager

Companion Device Manager และการเชื่อมต่อบลูทูธ เป็นตัวเลือกเพิ่มเติมในการเชื่อมต่ออุปกรณ์คู่ที่ไม่ได้ใช้ Wear OS

คุณอาจคุ้นเคยกับ Google Fit Android API ซึ่งรองรับการดำเนินการที่เกี่ยวข้องกับการออกกำลังกายหลายอย่าง เช่น การอ่านข้อมูลแบบเรียลไทม์และข้อมูลย้อนหลัง รวมถึงการบันทึกกิจกรรม เราได้ทำเครื่องหมายว่าเลิกใช้งานแล้วสำหรับ Android API ของ Google Fit หากแอปของคุณ ใช้ Google Fit Android API โปรดอ่านคำแนะนำในการย้ายข้อมูล เพื่อดูทางเลือกในการรักษาความสามารถของแอป การดำเนินการหลายอย่างที่คุณทำได้ด้วย Google Fit Android API สามารถทำได้ด้วย Android Health API เช่น คุณสามารถใช้ Recording API บนอุปกรณ์เคลื่อนที่แทน Google Fit Recording API ได้

หากต้องการวางแผนว่าจะใช้ API ใด คุณควรอ่านคำแนะนำเกี่ยวกับประเภทข้อมูลต่อไปนี้ เพื่อทำความเข้าใจข้อมูลที่คุณอาจอ่านได้จาก Health Connect และข้อมูลที่คุณอาจใช้ Health Services เพื่ออ่านจากอุปกรณ์ Wear OS

ประเภทข้อมูล Health Connect

ประเภทข้อมูลบริการด้านสุขภาพ

ประสบการณ์การใช้งานแบบหลายอุปกรณ์ที่สมบูรณ์ที่สุดจะใช้ทั้งบริการด้านสุขภาพและ Health Connect เพื่อมอบประโยชน์สูงสุดแก่ผู้ใช้

หมายเหตุ: หากต้องการอ่านข้อมูลประเภทใดประเภทหนึ่งจาก Health Connect กลับไปยังแอปของคุณ ผู้ใช้ต้องมีแอปอย่างน้อย 1 แอปในอุปกรณ์ที่เขียนข้อมูลประเภทนั้น ตัวอย่างเช่น หากต้องการใช้อัตราการเต้นของหัวใจในแอปบนอุปกรณ์เคลื่อนที่และแอปยังไม่ได้วัดอัตราการเต้นของหัวใจ คุณจะต้องอาศัยแอปอื่นที่เขียนค่าอัตราการเต้นของหัวใจไปยังที่เก็บข้อมูล Health Connect เพื่ออ่านข้อมูลอัตราการเต้นของหัวใจกลับ

ตัวอย่างเนื้อหาและโค้ดในหน้าเว็บนี้ขึ้นอยู่กับใบอนุญาตที่อธิบายไว้ในใบอนุญาตการใช้เนื้อหา Java และ OpenJDK เป็นเครื่องหมายการค้าหรือเครื่องหมายการค้าจดทะเบียนของ Oracle และ/หรือบริษัทในเครือ
